# 顺风关的携速石

**日期**：2026-07-08

**对应概念**：动量（Momentum）

---

## 故事

青岚山与雾谷之间，有一处唤作**顺风关**的隘口。关前坡面比缓坡关更刁钻：同一等高线上，常有浅洼、小丘、来回折返的羊肠径。许多学徒用好了步幅尺，却仍在中段徘徊——不是步子大小不对，而是**每跨一步便彻底停下**，下一步又从零开始判断，仿佛从未记得自己方才正沿同一方向走了好几程。

关里因此另设一司，专管**携速石**。

---

顺风关有三位携速吏，各守一亭。

**停步派**的吏姓顾，人称**顾零**。顾零发一块光滑石牌，上书：「每步独立，不携旧速。」学徒**阿砚**——曾在缓坡关跟陆惊学过缓降尺——持顾零的石牌下山，步子稳，却总在浅洼前停住：谷底明明在洼对岸，梯度却一时指不出明确方向，阿砚便立定不动，等下一阵雾散。有时一等便是半日。更糟的是，在窄谷里，梯度左右摇摆，阿砚依尺左跨一步、右跨一步，因**不保留任何前速**，轨迹呈锯齿，七日仍在同一等高线附近。

**猛冲派**的吏姓石，人称**石驰**。石驰的石牌不同：「既曾向下，便一路冲到底。」阿砚换石驰的牌，初时极快——连跨数步，借前速滚下山腰。可到了坡面转折处，前速仍朝旧方向猛冲，人便**冲过谷底**，撞上对坡又弹回；再冲，再弹，如球在碗里来回撞壁。石驰喝道：「速大才好破障！」阿砚却鼻青脸肿，比顾零还慢。

---

里正再请**陆惊**议规。陆惊这回未带新尺，只从袖中取出两枚**携速石**——卵石大小，一面刻纹，一面光滑。

「顾零与石驰，各对一半。」陆惊道，「顾零对的是：**不可让旧速喧宾夺主**；石驰对的是：**不可让旧速全无**。合宜之法，在**记一记、忘一部分、再与今步相合**。」

---

陆惊的规矩，名**携速三则**：

**其一，记向不记狂。** 每下一层，携速石记下**上一程的方向与力道**，但只保留**七成**——阿砚试之，连朝同一方向跨了三步，第四步不必从零起步，石上余速轻轻推他一把，浅洼前不再僵立。石驰问：「为何不是十成？」陆惊答：「十成便是盲冲；七成是**记得方向，却不被旧惯性绑架**。」

**其二，反向则消振。** 若新梯度与石上余速**相反**——窄谷里常见——余速不助冲，反**抵消**一部分反向跨步。阿砚在锯齿坡面试了七日，轨迹竟比顾零时平滑许多：左右摇摆时，旧速像一只手拽住反弹，振幅逐次缩小。顾零叹道：「我以为每步清空最稳，原来适度携速，反而减振。」

**其三，平处借推。** 坡面一时平坦、梯度微弱时，石上余速成了**暗推**——阿砚在浅洼底轻跨一格，余速接续，人滚过洼心，不必等雾散再判。陆惊说：「这与缓坡关的步幅尺不同：步幅管**这一步多大**，携速管**这一步是否承接上一步**。」

---

试石第十四日，里正验轨迹。

- **顾零牌**：锯齿多，浅洼前常停，**十四日未稳落谷底**；
- **石驰牌**：前段极快，后段在谷底附近**来回冲撞**，亦未安落；
- **携速石**：前段稳降，中段过洼不断，后段振幅缩小，**第十四日稳稳停在谷底标记**。

韩阔——缓坡关急跨派的步幅吏——也在座，问：「这与我的宽尺何异？不都是冲得快？」

陆惊摇头：「宽尺是**当下这一步大**；携速是**过去几步的方向叠加一小部分**。宽尺无记忆，故易过头；携速有记忆，却**只记七成、且遇反向则相消**——故既能借推过浅洼，又不会像石驰那样撞穿谷底。」

里正问：「七成可有定数？」

陆惊答：「常见在**五六成到九成**之间权衡。记太少，近乎顾零；记太多，近乎石驰。须与步幅尺、分批秤一并调——同一携速，步幅不同，结果亦异。」

里正问：「此法何名？」

陆惊在案上写下五字：

**携速循降**。

阿砚起初以为，说的是顺风关里滚石过洼的窍门。直到数年后他回关整理旧册，见陆惊所留图旁，另有一图：横轴是「迭代次数」，纵轴是「参数值」——一条曲线在**无携速**时于窄谷两侧锯齿震荡；一条在**携速过满**时冲过最优点再弹回；一条在**适度携速**时，振幅逐次衰减、轨迹顺滑入谷——他才忽然明白，关里记的，并不只是石头上的力道。

## 概念解释

### 基础：动量是什么？

在机器学习中，**动量（Momentum）** 是一种加速梯度下降收敛的优化技巧。它不只按当前梯度更新参数，还**保留一部分「上一次更新的方向与幅度」**，像给下山者一块「携速石」。

标准 **SGD with Momentum** 的更新规则为：

\[
\begin{aligned}
v_t &= \beta \cdot v_{t-1} + \nabla_\theta L(\theta_t) \\
\theta_{t+1} &= \theta_t - \eta \cdot v_t
\end{aligned}
\]

其中：
- \(v_t\)：**速度（velocity）**，累积的历史梯度方向（故事里的「石上余速」）
- \(\beta\)：**动量系数**，通常取 0.9（故事里的「保留七成」）
- \(\eta\)：学习率（步幅尺）
- \(\nabla_\theta L\)：当前梯度（当下的降向）

**梯度只反映「此刻坡面往哪斜」；动量反映「过去几步是否一直往同方向走」。** 二者叠加，更新更顺滑，也更容易「滚过」短暂平坦区。

故事里的对应关系：

| 故事元素 | 机器学习概念 |
|----------|--------------|
| 携速石 / 石上余速 | 速度向量 \(v_t\) |
| 保留七成 | 动量系数 \(\beta \approx 0.9\) |
| 顾零「每步独立」 | 普通 SGD（\(\beta = 0\)） |
| 石驰「一路冲到底」 | \(\beta\) 过大或有效步长过大 |
| 反向则消振 | 动量与当前梯度相反时相互抵消，减小震荡 |
| 平处借推 | 在梯度很小的区域，历史动量推动参数继续前进 |
| 携速循降 | 带动量的梯度下降（Momentum SGD） |

### 进阶：动量为什么有效？

1. **加速一致方向上的收敛**  
   若多个连续 batch 的梯度方向相近，动量会**累加**这些方向，等效于在「正确下坡方向」上步子更大——对应故事里连走三步后，第四步被余速轻推一把。

2. **抑制方向来回翻转的震荡**  
   在狭窄山谷（病态曲率、条件数大的损失曲面）中，普通 SGD 可能在两侧壁之间锯齿震荡。动量项在反向更新时会**抵消**部分步长，使振幅逐次缩小——对应「反向则消振」。

3. **帮助穿越平坦区**  
   当局部梯度接近 0（鞍点、浅洼底）时，仅靠当前梯度几乎不动；动量仍携带历史信息，推动参数继续移动——对应「平处借推」。

4. **与学习率的区别**  
   - **学习率 \(\eta\)**：这一步走多远（步幅尺）；  
   - **动量 \(\beta\)**：多大程度「记得」上几步的方向（携速石）。  
   二者独立调节，常配合使用。

### 深入：Nesterov 动量与更高级的优化器

1. **Nesterov Accelerated Gradient（NAG）**  
   俗称「**先望一步，再落脚**」：先按当前速度**预判**下一步位置，在该处计算梯度，再更新。直觉上，在快要冲过头时，能**提前刹车**。PyTorch 中 `SGD(momentum=0.9, nesterov=True)` 即此法。

2. **与 Adam 的关系**  
   Adam 等自适应优化器内部也使用**一阶矩估计**（梯度的指数移动平均），可视为带自适应学习率的动量类方法。但经典 Momentum SGD 在部分视觉任务上，配合良好调参仍极具竞争力。

3. **超参数建议**  
   - \(\beta\)：常用 0.9；批量较小时可试 0.99；  
   - 学习率：带动量时有时可比纯 SGD **略大**；  
   - 动量不替代学习率调度——近谷底仍宜收步（缓坡关篇）。

4. **何时效果最明显**  
   - 损失曲面狭长、曲率不均；  
   - 梯度方向噪声大但**长期趋势一致**（与小批量梯度下降联用）；  
   - 训练初期希望更快下降。

### 与已学概念的联系

- **梯度下降**（2026-06-25）：动量是梯度下降的扩展，引入速度状态。  
- **小批量梯度下降**（2026-07-06）：每个 batch 梯度有噪声，动量平滑噪声、突出主方向。  
- **学习率**（2026-07-07）：\(\eta\) 与 \(\beta\) 共同决定更新；只调 lr 不调 \(\beta\)，如同只带步幅尺不带携速石。  
- **反向传播**（2026-06-27）：反向传播提供 \(\nabla L\)，动量决定如何**累积**多次 \(\nabla L\)。  
- **过拟合与正则化**（2026-06-26、2026-07-01）：动量主要改善**优化过程**，不直接防止过拟合；过大有效步长仍可能导致训练不稳定。

### 实践建议

1. **对比实验：SGD vs SGD+Momentum**  
   固定学习率与 batch size，分别训练同一小网络，绘制 loss 曲线与参数轨迹（可用 2D 损失等高线图可视化），观察无携速时的锯齿与有动量时的平滑——对应顾零牌与携速石。

2. **调节 \(\beta\)**  
   试 `momentum=0.0, 0.5, 0.9, 0.99`，体会「记太少」与「记太多」（石驰牌）的差异。

3. **尝试 Nesterov**  
   ```python
   optimizer = torch.optim.SGD(model.parameters(), lr=0.01, momentum=0.9, nesterov=True)
   ```
   与 `nesterov=False` 对比验证集收敛速度。

4. **与 lr 调度联用**  
   动量 + `CosineAnnealingLR` 或 `StepLR`，理解「携速」与「收步」如何分工。
