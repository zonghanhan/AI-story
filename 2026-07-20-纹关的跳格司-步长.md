# 纹关的跳格司

**日期**：2026-07-20

**对应概念**：步长（Stride）与卷积/池化中的降采样

---

## 故事

阿砚在远目司学会「目域反推」之后，验帛坊里又挂出新匾：**跳格司**。里正说：「滑印逐格挪、缩格跳着盖，阵里有一数，远目司算目域时必用，却从无人专司——**每挪一步，印心向前几格**。」

阿砚不解：「印不是一格一格滑吗？」

跳格司主吏姓格，人称**格公**。格公在案上铺巨帛，长宽各十六格，取三格见方之印，自左上起演示：

「若**每印完一次，印心只前移一格**——上下左右皆然——小帛上会有**十四行十四列**个响应点；巨帛上相邻两响应，所盖之域**重叠两格**。」

他在相邻两响应之间用淡墨连出重叠带：「重叠多，纹理**细、稳**，但小帛**大**，后关算得慢。」

---

格公改令：「**每印完一次，印心前移两格。**」

同样巨帛、同样三格见方之印，小帛上只剩**七行七列**响应。阿砚数格：「响应少了一半。」

格公点头：「这叫**跳着盖**——不是印变小，而是**落印的位置变稀**。原帛上相邻两个响应，中间**空出一格没被选作落印中心**；两响应所盖之域，重叠只剩**一格**。」

里正问验关题：「十六格见方巨帛，三格见方之印，**不补边**——逐格挪与隔格跳，小帛边长各几？」

役吏按格公所立**跳格律**答：

- 逐格挪（跳 **1**）：\((16 - 3) / 1 + 1 = 14\)
- 隔格跳（跳 **2**）：\((16 - 3) / 2 + 1 = 7\)（整除，无余）

阿砚恍然：「跳格越大，小帛**缩得越快**——像缩格司合并格点，却是在**选落印中心**时就变稀了。」

格公纠正：「缩格是在**已有响应**上取尖合并；跳格是在**生成响应**时就少落几印。二者常并用，但**律不同**。」

---

第三日，关外来急帛：要在一炷香内验完，阵不能改印、不能减层。格公提议：「浅层仍逐格挪，保细纹；**第二层滑印改隔格跳**——小帛再减半，深关算量降四分之三。」

阿砚担心：「跳格之后，远目司反推，末层一格目及原帛更大，且**相邻响应之间**会不会漏看一条细裂？」

格公令目公同席。目公展溯源尺：「跳 **2** 不等于瞎半幅；是**用更少的哨兵**，每个哨兵仍带三格见方之目。漏检风险在**极细且恰落在两哨兵正中缝隙**的纹——故关外图说常令**浅层跳 1、深层跳 2**，或与缩格交替。」

里正定规：**跳格数**写入《纹关册》，与核大小、补边并列；验关必问「本层跳几格」。

---

又有一事。印匠问：「巨帛边长十七，三格印，隔格跳——小帛边长不是整数格。」

格公在帛角演算：\((17 - 3) / 2 + 1 = 8\)，余 **1** 格「拖尾」。「律言：**能整除则尽盖；不能整除则舍弃拖尾**——除非在巨帛四边**补空圈**（与远目司补格同类），把可盖范围凑成跳的整数倍。」

阿砚记：**补边 + 跳格 + 核大小** 三者一起决定小帛尺寸；只背「跳 2 就减半」不够，奇偶与余数都要算。

---

封关前，格公取两幅小帛并排：左幅**逐格挪**得十四格，右幅**隔格跳**得七格，同一深关继续滑印。深关输出：左 **12×12**，右 **6×6**。阿砚问：「为何深关差四格，不是差七格？」

格公：「深关的印也在跳；**父层小帛每缩一格，子层再按自己的跳格律缩一遍**。跳格是**每层各自**的——父层跳 2，子层仍可按跳 1 或跳 2，全关尺寸要**逐层乘算**。」

---

关外图师至，呈**《关外图说·跳格篇》**。图上卷积核在输入格上滑动，箭头标注 **stride = 1** 与 **stride = 2**：后者相邻两次落印中心相距两格，输出网格变稀；旁注公式

\[
H_{\text{out}} = \left\lfloor \frac{H_{\text{in}} - K + 2P}{S} \right\rfloor + 1
\]

阿砚对照跳格司里**逐格挪与隔格跳、小帛 14 与 7、浅层保细深层提速、补边消余数、逐层跳格相乘**——与图说 **Stride（步长）** 完全同构。

里正问：「此法何名？」

格公在案上写下二字：

**步长**。

阿砚这才明白：跳格司所管的，正是卷积与池化里**窗口每次移动的距离**；**步长为 1** 时响应最密，**步长大于 1** 时在卷积本身就做降采样，算得更快、特征更稀，并与远目司的目域、缩格司的合并一起，决定整关「看多细、看多快、看多远」。

---

## 概念解释

### 基础：步长是什么？

**步长（Stride，常记作 \(S\)）** 指：在卷积或池化操作中，**滑动窗口每次移动的距离**（以输入特征图像素/格为单位）。故事里的「跳格」即 stride。

- **Stride = 1**：窗口逐格移动（故事：逐格挪），输出特征图**空间尺寸最大**（在相同核大小、padding 下），相邻位置感受野**重叠多**。
- **Stride > 1**（常见为 2）：窗口隔若干格才落下一次（故事：隔格跳），输出**更稀疏、更小**，相当于在**该层**做降采样，计算量下降。

故事与概念的对应：

| 故事元素 | 机器学习概念 |
|----------|--------------|
| 印心前移几格 | Stride \(S\) |
| 逐格挪 | \(S = 1\) |
| 隔格跳 | \(S = 2\)（或更大） |
| 14×14 与 7×7 小帛 | 输出特征图高宽 |
| 跳格律公式 | 输出尺寸公式 |
| 拖尾舍弃 | 不能整除时的 floor / 截断 |
| 四边补空圈 | Padding \(P\) 与 stride 配合 |
| 浅层跳 1、深层跳 2 | 典型 CNN 设计：前层保分辨率，后层加速 |
| 与缩格并用 | Conv stride 与 Max Pool 都可降采样 |

**输出尺寸（单维，与故事一致）**：

\[
\text{out} = \left\lfloor \frac{\text{in} - K + 2P}{S} \right\rfloor + 1
\]

其中 \(K\) 为核大小，\(P\) 为对称 padding，\(S\) 为 stride。

**为何重要？**

1. **控制特征图大小**：不只有池化能缩小特征图；**stride=2 的卷积**同样减半（在常见设定下），且带可学习滤波。  
2. **速度与显存**：特征图越小，后续层计算越少。  
3. **感受野与采样**：stride 大 → 相邻输出点对应输入区域**重叠少**、采样更稀，与 [2026-07-19 感受野](./2026-07-19-纹关的远目司-感受野.md) 中的递推直接相关。  
4. **与「学习率步幅」区分**：[2026-07-07 学习率](./2026-07-07-缓坡关的步幅吏-学习率.md) 讲的是优化器在参数空间走多远；**步长（stride）** 讲的是卷积窗口在**空间网格**上跳多远——同名不同义。

**直觉**：把卷积核想成盖章——**步长就是每次盖完章，印泥中心在布上向前挪几格**；挪得越大，盖出的印越少，布上留下的「摘要」越粗、越省工。

### 进阶：步长与池化、现代结构

**卷积 stride vs 池化**

| | Stride 卷积 | Max/Avg Pooling |
|--|-------------|-----------------|
| 是否可学习 | 是（卷积核权重） | 池化无额外卷积参数（Avg 可视为固定核） |
| 降采样方式 | 线性组合后取位置 | 窗口内取最大/平均 |
| 故事 | 跳格选落印中心 | 缩格合并已有响应 |

实践中常**两者都用**（如 AlexNet、VGG 后期）；ResNet 等也常用 **stride=2 的 3×3 卷积** 代替「conv + pool」以简化结构。

**多层 stride 相乘**

每一层有自己的 \(S_l\)。从输入到第 \(l\) 层输出，**累积步长**（effective stride）约为 \(\prod_{i=1}^{l} S_i\)（与 kernel、padding 共同决定感受野，见感受野篇）。故事「父层跳 2、子层再跳」即**逐层相乘**，不能只看一层。

**转置卷积（Transposed Conv）**

上采样时「步长」表现为**插入空格再卷积**，是 stride 的逆过程（故事未展开，属进阶）；语义分割、生成模型常用。

**常见误区**

- 「stride=2 一定让边长精确减半」——仅当 \((H - K + 2P)\) 能被 2 整除时成立；否则要用公式或 `output_size` 算清楚。  
- 「stride 越大越好」——过大易**欠采样**，丢失细纹理与小目标；需与数据分辨率、任务匹配。  
- 混淆 **dilation（空洞率）** 与 stride：dilation 扩大核的**覆盖间隔**而不减少输出尺寸（同 stride=1 时）；stride 减少输出格点数。

### 深入：与已学概念的关系

1. **[卷积](./2026-07-13-纹关的滑印司-卷积.md)**  
   核大小决定「一次看多大」；**stride 决定「隔多远再看一次」**。

2. **[池化](./2026-07-18-纹关内的缩格司-池化.md)**  
   池化层也有 stride（默认常等于池化窗口大小）。Conv stride 与 pool stride 叠加，共同决定网络深度与分辨率曲线。

3. **[感受野](./2026-07-19-纹关的远目司-感受野.md)**  
   大 stride 使特征图变稀，且递推中**累积 stride 乘积**变大，深层一点对应输入上更宽区域。

4. **参数与计算量**  
   输出特征图边长约为原来的 \(1/S\)（理想整除时），该层及后续 FLOPs 大致按空间面积比例下降——与 [小批量梯度下降](./2026-07-06-筹巷里的分批秤-小批量梯度下降.md) 等训练效率话题间接相关。

5. **设计经验**  
   - 分类：早期 \(S=1\) 保留细节，后期 \(S=2\) + 加深通道。  
   - 检测/分割：高分辨率分支常保持 \(S=1\) 更久，避免小物体消失。
