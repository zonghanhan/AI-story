# 筹巷里的分批秤

**日期**：2026-07-06

**对应概念**：小批量梯度下降（Mini-batch Gradient Descent）

---

## 故事

青岚山下的药铺，有个专管进谷采药的外坊，唤作**筹巷**。巷中设一司，掌着全坊学徒的**秤法**——所谓秤法，便是每日根据当日的采药记录，微调学徒腰间的**步长绳**：绳上每结代表一步该跨多远。步长绳越准，学徒进雾谷时便越能循降而至，少踩泥沼，少绕假平地。

这秤法，与雾谷里的拾阶人阿禾所循者，本是同源。

---

筹巷里三位司秤的老手，各执一法，争了多年。

**全量派**的司正姓周，人称**周全**。周全的规矩极严：每日日落，须把坊内**全部**采药记录——三百名学徒、每人数十条路况笔记——统统抬进秤房，合账完毕，才算出当日的**降向**，再统一调整三百条步长绳。周全说：「降向须看全坊，少一条笔记，秤便歪。」

全量派的秤，公认最**准**。可也有弊端：合账常至三更，学徒多在秤房外等到睡着；且一旦某条笔记有误，整日三百人的步长绳都要重秤。坊间笑言：「周全司正秤一日，巷口堵三日。」

**单条派**的司正姓孟，人称**孟速**。孟速反其道而行：不抬整账，只从记录堆里**随手抽一条**，据这一条笔记定降向，改一条步长绳，然后立刻换下一条。三百学徒，一日可秤完三轮。孟速说：「雾谷日日变，等全账合齐，降向早过时。见一条，改一条，才叫活秤。」

单条派极**快**，学徒不必干等。但坊中常出怪事：同一学徒的步长绳，辰时秤完偏左，午时又偏右，未时竟指回起点——因为每条笔记只反映某一瞬、某一坑，降向随笔记剧烈摇摆。有学徒抱怨：「绳结一日三变，进谷比不进还乱。」

---

筹巷里正久欲定规，却始终不决。直到来了个暂管外坊的**陆惊**——验真镇惊信司出身，路过青岚，借宿筹巷。陆惊听罢两派争执，只问：「你二人可曾试过，一次秤**一小批**？」

周全道：「小批也是不全，与单条何异？」

孟速道：「小批也要合账，与全量何异？」

陆惊不辩，只请二人各借人手，用同一日记录试秤**同一批学徒**——坊中有个叫**小满**的学徒，步长绳近日总偏太急，进谷常踩泥。

---

第一日试秤，三种法子并行。

**周全**抬来三百份笔记，合至黄昏，降向指：步长绳**再紧半结**——全坊平均而言，最近步子仍偏大，须再小一点才稳。

**孟速**随手抽一条：小满昨日在东沟踩滑，笔记写「向东一步过猛」。孟速立刻把小满步长绳**松一结**——向东的步子该更大些。

**陆惊**则令书吏从三百份中**随机抽三十二份**，不挑不拣，合这「三十二份小账」定降向。三十二份里，东沟踩滑者七人，西槽缓降者九人，乱石假平地者若干——合账结果：步长绳**略紧四分之一结**，幅度介于周全与孟速之间。

小满当夜问陆惊：「究竟听谁的？」

陆惊答：「你明日再进谷。三法各试一日，连试五日，看哪条绳进谷最稳。」

---

五日过去，结果渐明。

**周全**的五日秤法，步长绳每日一变，但变幅极小；小满进谷确实稳，少绕圈，可每夜须在秤房外等到三更。全坊三百人，五日只完整秤了五轮——对变化快的雾谷，有时降向已滞后。

**孟速**的五日秤法，小满步长绳一日数变：有时紧、有时松、有时东、有时西。小满进谷如喝醉酒，草图上圈圈比阿禾初进谷时还乱。可孟速五日里把三百人各秤了十余轮——**更新次数**远胜周全。

**陆惊**的五日秤法，每次随机三十二份，一日秤**六轮**——比周全密，比孟速稳。小满步长绳每日变三四次，变幅不大不小；进谷时偶有摇摆，却不再像孟速法那般骤转。更奇的是：陆惊每轮抽的三十二人**不重样**，五日后，近九成笔记都被抽到过至少一次——既非全量那般等齐，又非单条那般以偏概全。

---

第六日，里正召集两派。孟速仍不服：「三十二份也是抽样，凭什么比我的单条更准？」

陆惊令书吏取来五日记录，做一图：横轴是「每次秤法所用的笔记份数」，纵轴是「小满进谷绕圈次数」。

- 全量三百份：绕圈 **2** 次，但每日仅 **1** 轮更新；
- 单条 **1** 份：绕圈 **11** 次，每日 **15+** 轮更新；
- 小批 **32** 份：绕圈 **4** 次，每日 **6** 轮更新。

陆惊道：「周全秤的是**全坊平均降向**，准，但慢，且一错便拖累三百人。孟速秤的是**一条笔记的降向**，快，但一条只代表一坑一瞬，方向噪声太大。我秤的是**一小批随机笔记的平均降向**——一批三十二，既抹平单条的偶然，又不必等三百齐；一日多轮，降向跟得上雾谷变化。」

周全问：「三十二这个数字，可有讲究？」

陆惊答：「无绝对讲究，只有权衡。批太小，接近孟速，噪声仍大；批太大，接近周全，又要久等。三十二、六十四、一百二十八，都是坊里常试之数——须能一次抬进秤房、一轮合账不致三更，且含够多样路况，让平均降向**大致可信**。」

里正问：「此法何名？」

陆惊在案上写下四字：

**分批循降**。

小满起初以为，说的是筹巷秤步长绳的规矩。直到数年后他回坊整理旧册，见陆惊所留图旁，另有一图：横轴是「每次更新所用的样本数」，纵轴是「损失曲线震荡幅度」——曲线在 **1** 处极高，在 **300** 处更新极疏，在 **32** 附近又稳又密——他才忽然明白，筹巷里称的，并不只是绳结。

## 概念解释

### 基础：为什么需要「分批」？

在机器学习中，模型训练要不断**更新参数**，使损失函数变小。每次更新，都需要先算**梯度**——而梯度来自训练数据。

有三种常见做法：

| 方法 | 每次用多少数据 | 特点 |
|------|----------------|------|
| **批量梯度下降（BGD）** | 全部训练集 | 梯度最准，但每轮极慢、占内存大 |
| **随机梯度下降（SGD）** | 1 个样本 | 更新极快，但梯度噪声大、训练震荡 |
| **小批量梯度下降（Mini-batch GD）** | 一小批（如 32、64、128） | 兼顾速度与稳定性，**深度学习默认选择** |

故事里的对应关系：

- **周全 / 全量派** → 批量梯度下降（Batch GD）
- **孟速 / 单条派** → 随机梯度下降（SGD）
- **陆惊 / 分批循降** → 小批量梯度下降（Mini-batch GD）
- **步长绳** → 模型参数
- **降向** → 梯度方向
- **采药笔记** → 训练样本

### 进阶：小批量为何有效？

1. **噪声与速度的平衡**  
   单个样本的梯度往往偏离「全体平均方向」；全量梯度最准但算一次代价高。小批量取随机若干样本求平均，既比单样本稳定，又比全量快得多。

2. **随机性带来泛化**  
   每轮随机抽取不同 mini-batch，相当于在优化路径上加入适度扰动，有助于跳出浅层局部最优，与故事里的「随机抽三十二人、五日后近九成笔记都被抽到」同一道理。

3. **硬件并行**  
   GPU 擅长对固定大小的张量做并行矩阵运算。batch size 为 32、64 等时，硬件利用率高；batch size 为 1 时并行度差，batch size 为全量时常超出显存。

4. **与 epoch 的关系**  
   - **1 个 epoch** = 整个训练集被用过一遍（在 mini-batch 设定下，通常要多个 batch 才覆盖全集）。  
   - 故事里的「一日六轮、五日近九成笔记被抽到」，对应训练中多个 epoch、每个 epoch 内多次 batch 更新。

### 深入：batch size 怎么选？

没有 universal 最优值，需在以下因素间权衡：

- **较大 batch（如 128、256）**：梯度估计更稳，训练曲线更平滑，但每步耗时更长，有时泛化略差，且需更大显存。
- **较小 batch（如 16、32）**：更新更频繁、正则化效应更强，但噪声大，可能需要更小心的学习率。
- **常见起点**：32 或 64；视觉任务常用 32–128；大模型受显存限制可能用 gradient accumulation 模拟更大 batch。

与「雾谷拾阶人」中**步长（学习率）**的关系：batch size 变大时，梯度更准，有时可略增大学习率；batch size 变小时，噪声大，常需减小学习率——二者需配合调节。

### 与已学概念的联系

- **梯度下降**（2026-06-25）：小批量梯度下降是其在工程上的标准实现方式。  
- **反向传播**（2026-06-27）：每个 mini-batch 前向计算一次、反向传播一次、更新一次参数。  
- **损失函数 / 交叉熵**（2026-06-28、2026-07-05）：每个 batch 上计算 batch 平均损失，再对参数求梯度。  
- **过拟合与泛化**（2026-06-26）：适度的 batch 噪声有时有助于泛化；过大 batch 需配合其他正则手段。

### 实践建议

1. 用 PyTorch 或 TensorFlow 加载 MNIST / CIFAR-10，分别设置 `batch_size=1`、`32`、`全量`，对比：  
   - 每 epoch 耗时  
   - 损失曲线是否震荡  
   - 验证集准确率  

2. 观察 `DataLoader` 中 `shuffle=True` 的作用——对应故事中「每轮随机抽三十二份、不重复覆盖全集」。

3. 在 TensorBoard 或 wandb 中绘制 train loss，比较不同 batch size 下的曲线形态。
