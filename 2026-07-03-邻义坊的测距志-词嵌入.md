# 邻义坊的测距志

**日期**：2026-07-03

**对应概念**：词嵌入（Word Embedding）

---

## 故事

邻义坊不靠河，靠字吃饭。坊中各家商铺、行当、节庆称谓，都须登记在**义理署**的**字据册**上——册上每一词，代表坊里的一种说法：「雨具」「伞」「遮雨」「借伞」「还伞」……义理署不判这些词孰是孰非，只管一件事：**给每个词定一个位置**。

这位置不在地图上，而在一张看不见的多维**义理图**里。署老顾先生称它为**测距志**：词与词之间，不比笔画多少，不比读音轻重，只比**相距多远**。

---

学徒方照入署时，顾先生递给他一本空白册，册上只有规矩，没有坐标：

> **同境共现者，相距当近；异境常别者，相距当远。**

方照不解：「境在何处？距又如何量？」

顾先生不答，只领他去做**填境**——义理署每日收到坊民写来的**短笺**，每笺五到十个词，是真实语境里截下的一小段话：「明日有雨，记得带**伞**」「**伞**骨断了，去王铺修」「借你**伞**，三日后**还**」。短笺里词与词同现，便算**共境**；从未在同一段话里出现过的，便算**异境常别**——至少在当前所见资料里如此。

方照的工作，起初像孩童游戏：**猜邻词**。署里随机遮去短笺中的一个词，让他根据前后词猜被遮的是哪一个。猜错了，顾先生便微调被遮词与邻词在义理图上的相对位置——猜「伞」时若写成「桌」，则把「桌」推远，把「伞」拉近「雨」「遮」一侧；猜对了，亦要微调，因同笺中尚有更贴切的词。

方照问：「我们从未见过义理图，如何微调？」

顾先生指墙上一排**滑轮绳**：每个词对应一根绳头，绳头牵动图上一个不可见的点；**猜邻**时，所有被牵动的点同时挪一点——不是只挪一个词，而是**凡与这次失误相关的词，都挪**。

---

数月后，方照发现一件怪事：他从未被教过「伞」与「雨具」何意，可在义理图上，两词**自动靠拢**——因坊里短笺里它们常在同境出现；「伞」与「桌」则远，因共境极少。

更有奇者：「君王」「男子」「女子」「王后」四词，本各属不同短笺体系，却在图上形成一组**平行四边形**——

「君王」到「男子」的距，与「王后」到「女子」的距，几乎相等；  
「君王」到「王后」的距，与「男子」到「女子」的距，亦几乎相等。

方照在测距志上标出这四点，连线的形状像一方**挪移的框**：若在「君王」处加一根向量，指向「男子」，再把这个向量**平移**到「王后」所在，箭头尖端竟落在「王后」旁——若再沿「女子」方向微调半寸，竟抵「王后」本身。

他跑去问顾先生：「词义难道可以**做算术**？」

顾先生答：「你做的不是词义算术，是**距的算术**。图上的距，是共境千次、万次**填境**与**猜邻**磨出来的；距近，不代表同义，而代表**可互相替换的语境相似**。」

---

邻义坊外乡来一位**编号司**的吏员，姓陆，携一册**编号表**：每个词配一个整数，「伞」三百一十二，「雨」四百零五，「桌」一千零二……陆吏员主张：「义理署何必画 invisible 的图？整数**唯一、好存、好算**，我编号司才是正途。」

顾先生不辩，只请陆吏员做同一题：从「君王 − 男子 + 女子」推出该对应何词。

陆吏员按整数算：三百减一百加九十，得二百九十；查表，二百九十是「**盐**」——因编号按部首笔画与入册先后而定，与语境无关。

方照按义理图做向量：从「君王」指向「男子」的向量，平移到「女子」，尖端落在「王后」。

里正当场验短笺：「**王后**出席春祭，**男子**不得入内」——「王后」与「男子」的语境关系，恰与「君王 / 女子」成对。编号表给出「盐」，义理图给出「王后」。

陆吏员不服：「你们磨图磨了千张短笺，我只须按规则编号一次。」

顾先生道：「编号表把词当**孤号**；测距志把词当**共境中的点**。孤号无法回答『遮去一词，邻境中谁最可能填上』——而**猜邻**，正是坊里修伞、写借据、办节庆时天天在做的事。」

---

义理署内部分两派，与编号司之争不同。

**广境派**主张：一张短笺里，每个词都与其余所有词两两算距——「明日」「有雨」「记得」「带」「伞」，五词共境，则两两拉近。好处是**信息用得足**；坏处是「明日」与「伞」本无直接关联，也被拉近，图上的距**发虚**。

**狭境派**主张：只把**紧挨着的邻词**当共境——「带」与「伞」拉近，「明日」与「有雨」拉近，但不强迫「明日」与「伞」相近。好处是**距更干净**；坏处是远词之间的间接关系，要靠许多短笺**慢慢传**。

方照两种都试，发现狭境派在「猜邻」上更准；广境派在短笺极长时略占上风，但邻义坊的日常短笺多为五至十字，署里遂以狭境为主，广境为辅。

顾先生称狭境法为**邻窗填境**，广境法为**全笺填境**——名不同，理同一：**用共境猜词，用猜词磨距**。

---

有一日，坊外来人问：「『苹果』在你们图上，离『果』近，还是离『公司』近？」

方照查图：在邻义坊收到的短笺里，「苹果」多与「果」「甜」「红」共境，极少与「公司」同现——故「苹果」离「果」极近，离「公司」极远。

来人又问：「若另一城邦的短笺里，『苹果』常指那家叫 Apple 的商号，与『公司』『股票』共境呢？」

方照愣住。顾先生接话：「测距志**无唯一真理图**，只有**依短笺而定的图**。邻义坊的图，回答邻义坊的境；彼城的图，须用彼城的笺去磨。距不是词里自带的，是**语境灌进去的**。」

方照由此明白：义理图上的「近」，不是词典里的同义，而是**在某一堆真实用法里，可互相顶替的近似**。

---

季末，里正考核：给方照一百张**从未入署**的新短笺，遮去每笺一词，比义理署与编号司谁猜中最多。

编号司靠整数，无法利用邻词，只能瞎蒙或查同字——百笺中十二；方照靠义理图，百笺中七十一。

陆吏员仍说：「七十一而已，不是百。」

顾先生指测距志封面上的四字——**词有坐标**——又指署规：

> **坐标不显于义，显于共境；距可加减，故可推词。**

方照这才贯通：每个词在义理图上的位置，即是一串**隐藏的数**——数本身无义，**数与数之间的距**却编码了共境千次的信息；「君王 − 男子 + 女子 ≈ 王后」不是魔法，是距在向量空间里**可平移**；编号司的整数不能做这个，因整数与共境无关。

他抬头问：「这测距志，在署外叫什么？」

顾先生在册末写下二字：**词嵌**——词，嵌进可测距的空间；空间由填境与猜邻**学**来，不由人手工指定。

方照合上册子。邻义坊的字依旧各指各的物，可在义理图里，它们终于不再是一行孤号，而是**彼此有距、可猜可推**的坐标点——机器若要学会「像坊民那样选词」，须先学会这张由共境磨出的图。

---

## 概念解释

这个寓言讲的是自然语言处理中的**词嵌入（Word Embedding）**——把每个词表示成低维连续向量，使**语义或用法相近**的词在向量空间中**距离相近**；向量还可做加减运算，对应著名的 **king − man + woman ≈ queen** 类比。

### 基础：词嵌入是什么？

| 寓言元素 | 机器学习含义 |
|---------|-------------|
| 义理图 / 测距志 | **词向量空间（Embedding Space）** |
| 每个词的位置 | **词向量（Word Vector）**，如 300 维浮点数 |
| 填境 / 短笺 | **训练语料（Corpus）** 中的上下文 |
| 猜邻 | **自监督学习**：用上下文预测词（或反过来） |
| 同境共现 → 相距近 | **分布假设（Distributional Hypothesis）**：上下文相似的词，语义相近 |
| 滑轮绳同时微调 | **嵌入层参数随训练一起更新** |
| 编号司整数表 | **One-Hot 编码**——高维、稀疏、词与词等距，无法表达相似度 |
| 邻窗填境 | **Skip-gram**——用中心词预测邻词 |
| 全笺填境 | **CBOW（Continuous Bag of Words）**——用周围词预测中心词 |
| 君王 − 男子 + 女子 ≈ 王后 | **词向量线性类比（Linear Analogy）** |
| 不同城邦不同图 | **嵌入依赖训练语料**，无跨域 universal 的唯一坐标 |

**核心思想**：机器无法直接「理解」词义，但可以从**大量文本里词如何共现**中，学出每个词的向量表示；向量间的**余弦相似度**或**欧氏距离**反映用法上的接近程度。

### 由浅入深

**第一层（本科基础）：为何不用 One-Hot？**

One-Hot 把「伞」编为 `[0,0,…,1,…,0]`，「雨具」为另一维度的 1——两向量**正交**，距离相同，无法表达「伞与雨具比伞与桌更相近」。词嵌入把每个词压到 50～300 维的**稠密向量**，相似词自然靠近，且参数量可训练。

**第二层：如何学出向量？——Word2Vec**

2013 年 Mikolov 等人的 **Word2Vec** 是本科最常讲的词嵌入方法，两种架构对应寓言两派：

| 方法 | 寓言 | 做法 |
|------|------|------|
| **Skip-gram** | 邻窗填境 | 给定中心词，预测窗口内邻词 |
| **CBOW** | 全笺填境 | 给定周围词，预测中心词 |

训练目标本质是**分类问题**：在词表上万词中，猜哪个词最可能出现——输出层常用 **Softmax**（本系列后续可专讲）。通过反向传播（见 2026-06-27 寓言），**嵌入矩阵**中的行向量被不断更新，最终每个词得到一行向量。

**第三层：向量算术与语义**

训练良好的嵌入空间中，方向向量可编码**关系**：

\[
\vec{v}_{\text{king}} - \vec{v}_{\text{man}} + \vec{v}_{\text{woman}} \approx \vec{v}_{\text{queen}}
\]

类似地：「巴黎 − 法国 + 意大利 ≈ 罗马」。这不是说词义可以任意做加减，而是**某些线性方向对应「性别」「首都-国家」等关系**——前提是训练语料中这类例子足够多。

**第四层（进阶）**

| 主题 | 说明 |
|------|------|
| **GloVe** | 基于全局共现矩阵分解；与 Word2Vec 殊途同归 |
| **FastText** | 子词（subword）嵌入，可处理未登录词 |
| **上下文相关嵌入（ELMo、BERT）** | 同一词在不同句中向量不同——「苹果」在果篮句与股票句中位置不同，对应顾先生「不同城邦不同图」 |
| **嵌入作为神经网络第一层** | 在 NLP 模型中，嵌入层权重可预训练后微调（Fine-tuning） |
| **向量空间可视化** | t-SNE / PCA 将高维压到 2D 作图，直观看到聚类 |

### 与本系列已学概念的衔接

| 已学概念 | 与词嵌入的关系 |
|---------|---------------|
| **神经网络**（2026-06-29） | 嵌入层是 NLP 网络的第一层；猜邻即前向 + 损失 + 反向传播 |
| **损失函数**（2026-06-28） | 猜邻错词 → 交叉熵等损失 → 驱动嵌入更新 |
| **反向传播**（2026-06-27） | 梯度回传到嵌入矩阵，调整每个词的向量 |
| **梯度下降**（2026-06-25） | Word2Vec 用负采样等技巧高效优化 |
| **过拟合与泛化**（2026-06-26） | 语料过小则嵌入只记住局部共现，类比关系弱 |
| **正则化**（2026-07-01） | 可对嵌入向量加约束，防止过拟合罕见词 |

### 常见误区

1. **「向量近 = 同义词？」**  
   近表示**用法/语境相似**，「伞」与「雨具」近，但「「伞」与「折叠」也可能近——未必同义。

2. **「词嵌入等于理解语言？」**  
   只是**从统计共现中学到的表示**；没有因果、常识、推理，需与更强大模型结合。

3. **「类比恒成立？」**  
   king − man + woman 是经典例子，并非对所有词对都成立；依赖语料规模与领域。

4. **「一套嵌入通吃所有任务？」**  
   金融语料与童话语料训出的「苹果」向量不同——**域适应**很重要。

## 学习资料与了解途径

### 入门（本科基础）

1. **吴恩达《深度学习专项》序列模型（Sequence Models）周**  
   - 讲解 Word2Vec、负采样、GloVe 直觉。  
   - 搜索：`Andrew Ng Word2Vec`

2. **Jay Alammar《The Illustrated Word2vec》**  
   - 网址：https://jalammar.github.io/illustrated-word2vec/  
   - 图文并茂解释 Skip-gram、CBOW、负采样；与「猜邻」「邻窗填境」高度对应。

3. **3Blue1Brown《Word Embeddings》相关视频**（或 NLP 入门合集）  
   - 向量空间直觉、类比可视化。

### 经典论文与工具

4. **Mikolov et al., «Efficient Estimation of Word Representations in Vector Space» (2013)**  
   - Word2Vec 原始论文；提出 Skip-gram 与 CBOW。  
   - arXiv：https://arxiv.org/abs/1301.3781

5. **Mikolov et al., «Distributed Representations of Words and Phrases and their Compositionality» (2013)**  
   - 负采样（Negative Sampling）与词组向量。  
   - arXiv：https://arxiv.org/abs/1310.4546

6. **Pennington et al., «GloVe: Global Vectors for Word Representation» (2014)**  
   - 共现矩阵视角；https://nlp.stanford.edu/projects/glove/

7. **gensim 库 Word2Vec 教程**  
   - 网址：https://radimrehurek.com/gensim/auto_examples/tutorials/run_word2vec.html  
   - 用 Python 在自有语料上训练嵌入，复现「猜邻」实验。

### 教材章节

8. **《统计自然语言处理》（宗成庆）词向量章节**  
   - 中文教材，系统介绍 Word2Vec、GloVe 与评估方法。

9. **《深度学习》（花书）第12章「应用」— NLP 部分**  
   - 嵌入层在神经网络中的形式化定义。

10. **Speech and Language Processing（Jurafsky & Martin）第6章 «Vector Semantics and Embeddings»**  
    - 免费在线第三版草稿：https://web.stanford.edu/~jurafsky/slp3/  
    - 本科 NLP 经典参考，分布假设讲得很清楚。

### 实践建议

- 用 **gensim** 或 **PyTorch nn.Embedding**，在小语料（如邻义坊式短句集合）上训练 Skip-gram，打印与「伞」最近的 10 个词。  
- 手动验证类比：`most_similar(positive=['君王','女子'], negative=['男子'])`（换成英文 king/woman/man 在英文语料上更常见）。  
- 用 **t-SNE** 将 300 维压到 2D，观察「雨具 / 伞 / 桌」是否聚类符合直觉。  
- 对比 **One-Hot** 与 **Embedding** 在「猜邻」任务上的参数量与准确率——理解编号司为何输给义理署。  
- 进阶：加载 **预训练中文嵌入**（如 Tencent AI Lab Embedding、中文 Word2Vec），观察「苹果」在新闻语料 vs 童话语料中的近邻词差异。

### 延伸阅读方向

- **子词嵌入（BPE、WordPiece、FastText）**：解决未登录词与形态变化。  
- **上下文嵌入（BERT、GPT）**：从「一词一向量」到「一词多向量」。  
- **句向量与文档向量**：从词嵌入到 Sentence-BERT。  
- **嵌入中的偏见（Bias）**：训练语料中的性别、种族偏见会进入向量——社会与技术交叉议题。  
- **知识图谱嵌入（TransE 等）**：关系型「测距志」，与词嵌入互补。
