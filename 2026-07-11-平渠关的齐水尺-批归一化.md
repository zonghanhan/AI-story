# 平渠关的齐水尺

**日期**：2026-07-11

**对应概念**：Batch Normalization（批归一化）

---

## 故事

青岚山下行，过了禁阵关，又逢一关，唤作**平渠关**。关内仍是一座**传灯阵**——百根灯柱、细线相连、阵心一块验灯石——阿砚持矩司合降、散役抽签诸法至此，欲再训此阵识得关外行人衣色。里正给他五千张样图，命他旬日内训毕。

初五日，阿砚便觉不对。

阵并非不聪明——样图上的误差在降；可每训过**一批**图（筹巷里唤作「一小批」），下一批图一进阵，阵中**水渠**（阿砚后来才知，那水渠便是层与层之间的信号）便忽涨忽落：有时某段渠水**漫过堤岸**，验灯石亮得刺眼；有时又**几近干涸**，石面纹丝不动。矩司合降的步子（学习率）稍一大，阵便震荡；稍一小，又数日不动。阿砚试过散役抽签、绳墨松紧，新图泛化尚可，**训阵之稳**却始终像走钢丝。

里正来验，皱眉道：「阵在学，可**内渠之水**日日改色改位。后一层所接之水，与昨日同批不同；前一层刚调好，后一层面对的已是另一番水位。此谓**内渠漂移**——你调的是昨天的阵，训的却是今天的水。」

---

关内因此另设一司，专管**齐水**。

主吏姓渠，人称**渠公**。渠公不碰矩账，也不摇签筒，只持一把**齐水尺**，尺分三段：一段量**均位**（均值），一段量**散幅**（方差），一段名**闸规**（可调的尺度与基线）。渠公规矩听来平淡，却极苛刻：

「**每一批图过阵，每段水渠在进下一层灯柱之前，须先齐水。**」

阿砚问：「如何齐？」

渠公引他到阵侧一条**明渠**前。明渠透明，可见水流。只见仆役将本批一百张图依次送入，水流涌入第一段渠；渠公将齐水尺探入，读得：「此段水**偏高三寸、起伏散乱**。」随即下令：

**其一，先定均位。** 以本批水流之**平均水位**为尺，令整段水**先降至均位线**——高的减、低的加，使此批水「以均位为零」。

**其二，再定散幅。** 量此批水相对均位之**起伏幅度**；令散幅归一——起伏大的压、起伏小的放，使此批水「散幅如一」。

**其三，再施闸规。** 均位与散幅齐了之后，并不就此固定死；渠公又拨动闸规上两枚旋钮——一枚名**开度**（\(\gamma\)），一枚名**基线**（\(\beta\)）——令齐水后的水流再按阵法需要**可宽可窄、可高可低**。阿砚问：「既已齐至均位散幅，为何还要开度与基线？」渠公道：「齐水是**先稳住本批之水**，开度基线是**再让阵自己决定**要宽渠还是窄渠、基线要高还是低。尺是规矩，闸是学问。」

阿砚恍然：齐水不是把阵训成死水一潭，而是**每批先归一，再可学**。

---

阿砚依渠公之法，在每两层灯柱之间的水渠上各设**齐水尺站**。训阵流程变为：

样图入阵 → 前向传信 → **至水渠则齐水**（本批算均位散幅，归一，再经开度基线）→ 下一层灯柱 → … → 验灯石 → 误差回传 → 矩司合降。

初十日再验，震荡大减。阿砚把学习率由「缓坡关」的细步改为稍大步，阵仍稳。里正问何故，渠公道：「**后层不必再猜前层今日水位几何**——每批进来，水先齐过再流。前层动、后层接到的却是**尺度一致**之水，矩司合降便敢迈步。」

---

关内亦有争论。

禁阵关的散公来访，见齐水尺站，笑问：「我散役是逼柱**不可共倚**；你齐水是逼水**不可乱漂**。可曾冲突？」渠公道：「散役在**灯柱之后**，齐水在**水渠之中**——先齐水，再传柱，再抽签熄柱，次序分明。」阿砚试了三法：

- **仅矩司合降**：内渠漂移，震荡；
- **矩司 + 齐水**：训得稳、收敛快；
- **矩司 + 齐水 + 散役**：新图泛化佳，训阵亦稳。

绳墨派韩阔亦至，道：「松紧律是限**线之粗细**；齐水是限**水之齐散**——线可细，水仍可能一日一貌；故绳墨与齐水，各管一域。」

---

第三旬，阿砚几乎训毕，却生一惑。

里正命他将阵**封关实守**——不再喂样图，只接关外行人，实时判衣色。阿砚按训阵之法，仍令仆役每过**一批**行人便齐一次水；里正摇头：「关外行人**一来一张**，哪有『本批』？」

渠公此时取出**第二把尺**——色暗，尺背刻「**历年汇尺**」。渠公道：「**训阵之时**，齐水尺量的是**眼前这一批**的均位与散幅——批与批之间，水纹本就不同，须当场量、当场齐。 **封关实守之时**，不能再等凑批；须用历年汇尺——训阵过程中，渠司已将每段水渠的均位、散幅**逐批累记、加权汇算**，封关后以此**固定之尺**齐水，不再随单张图剧变。」

阿砚问：「若封关后仍用本批之尺，会如何？」渠公答：「单张图视为一批，均位散幅噪声极大，阵会**随人一抖**——今日齐法，明日便废。」

阿砚记下两句：

- **训时齐本批之水**；
- **用时齐历年汇尺**。

---

旬末，里正以八百新图再验，验灯石准而稳。里正问：「此法何名？」

渠公在案上写下四字：

**批内齐水**。

阿砚起初以为，平渠关不过是于层间水渠设尺，令每批水流先齐均位、再齐散幅，最后用开度基线微调，好让后层不必猜前层今日水位。直到数年后他赴外郡，见一册**《深度训阵图说》**，其中一页画层间方框，注「\(\mu_B, \sigma_B^2\)」「\(\hat{x} = \frac{x-\mu_B}{\sqrt{\sigma_B^2+\epsilon}}\)」「\(y = \gamma \hat{x} + \beta\)」「running mean/var」，并写「置于 Conv/FC 与激活之间」——他才忽然明白，齐水尺量与齐的，并不只是关内明渠里那一批水。

## 概念解释

### 基础：Batch Normalization 是什么？

**Batch Normalization（批归一化，常简称 BN 或 BatchNorm）** 是训练深度神经网络时用来**稳定各层输入分布**的一种技术。其核心思想：

> 对**每一个 mini-batch**，在网络的某些层（通常是全连接层或卷积层**之后、激活函数之前或之后**，依实现而定），将该层激活值在 batch 维度上**归一化**到近似均值 0、方差 1，再通过**可学习的缩放 \(\gamma\) 与平移 \(\beta\)** 恢复表达能力。

对某一层的激活 \(x\)，在一个 mini-batch 上计算均值 \(\mu_B\) 与方差 \(\sigma_B^2\)，则：

\[
\hat{x} = \frac{x - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}, \quad y = \gamma \hat{x} + \beta
\]

其中 \(\epsilon\) 为防除零的小常数；\(\gamma, \beta\) 与权重一样通过反向传播学习。

故事里的对应关系：

| 故事元素 | 机器学习概念 |
|----------|--------------|
| 平渠关 / 层间水渠 | 神经网络相邻层之间的激活传递 |
| 内渠漂移 | Internal Covariate Shift（内部协变量偏移）：训练中各层输入分布持续变化 |
| 齐水尺量均位 | 对 mini-batch 计算均值 \(\mu_B\) |
| 齐散幅 | 对 mini-batch 计算方差 \(\sigma_B^2\) 并归一化 |
| 开度 / 基线 | 可学习参数 \(\gamma\)（scale）与 \(\beta\)（shift） |
| 训时齐本批之水 | 训练阶段使用**当前 batch** 的统计量 |
| 历年汇尺 | 推理阶段使用训练时累积的 **running mean / running variance** |
| 批内齐水 | Batch Normalization |

**直觉**：深度网络训练时，前面层的参数更新会改变后面层「看到」的输入分布，使后层不断适应移动的目标，训练变慢、对学习率敏感。BN 在每一层把输入**先拉回稳定尺度**，后层面对的是**分布更一致**的输入，梯度更稳、收敛更快——对应「后层不必再猜前层今日水位」。

### 进阶：BN 为什么有效？

1. **缓解内部协变量偏移（Internal Covariate Shift）**  
   Ioffe & Szegedy 原始论文的动机：训练过程中，随着前面层参数变化，后面层输入分布不断漂移。BN 对每个 batch 做归一化，使层输入的**一阶、二阶矩**稳定，减轻这种漂移。

2. **\(\gamma\) 与 \(\beta\) 不是多余的**  
   若只做归一化，可能限制网络表达能力（例如把有用的幅度信息抹平）。可学习的 \(\gamma, \beta\) 允许网络在「需要时」恢复非零均值、非单位方差——对应「尺是规矩，闸是学问」。

3. **允许更大学习率、加速收敛**  
   BN 使损失 landscape 更平滑，实践中常能使用更大学习率，训练轮数更少——对应阿砚在齐水后可「稍大步」仍稳。

4. **轻微正则化效果**  
   每个 batch 的 \(\mu_B, \sigma_B^2\) 含噪声，引入轻微随机性，有时可减轻过拟合；但 BN **不是**主要正则手段（Dropout、权重衰减才是）。二者可并用，需注意层序与 `train`/`eval` 模式。

5. **训练与推理模式不同**  
   - **训练**：用当前 mini-batch 统计量，并**指数移动平均**更新 running mean/var；  
   - **推理**：用训练全程累积的 running 统计量，**不再依赖 batch**——对应「训时齐本批，用时齐历年汇尺」。  
   忘记切换 `model.eval()` 是常见 bug。

### 深入：放置位置、变体与使用注意

1. **放在哪一层？**  
   经典做法：Conv/FC → BN → ReLU（或 FC → BN → 激活）。也有「激活后再 BN」的实现；关键是与所用框架、论文设定一致。现代 ResNet、CNN、早期 CV 模型广泛采用 **Conv-BN-ReLU**。

2. **Batch 大小的影响**  
   BN 统计量来自 batch；**batch 过小**（如 1–2）时估计噪声大，效果差。极小 batch 常用 **Group Normalization、Layer Normalization** 等不依赖 batch 维度的归一化——Transformer 中主流是 **LayerNorm**，而非 BN。

3. **与 Dropout 的关系**  
   二者常同现于全连接网络。经验上需注意顺序与模式切换；CNN 中 BN 极常见，Dropout 相对较少。参见 [2026-07-10 禁阵关的散役司 — Dropout](./2026-07-10-禁阵关的散役司-Dropout随机失活.md) 中的「与 Batch Normalization 的关系」。

4. **主要变体**  
   - **Layer Normalization（LN）**：在特征维度归一化，不依赖 batch，用于 RNN/Transformer；  
   - **Instance Normalization（IN）**：常用于风格迁移；  
   - **Group Normalization（GN）**：小 batch 场景的折中。  

5. **与已学概念的联系**  
   - **学习率**（2026-07-07）：BN 常允许更大步长；  
   - **小批量梯度下降**（2026-07-06）：BN 的「批」即 mini-batch 统计；  
   - **过拟合与泛化**（2026-06-26）：BN 有轻微正则，但主要贡献是**稳定训练**；  
   - **神经网络**（2026-06-29）：作用于层间激活；  
   - **反向传播**（2026-06-27）：\(\gamma, \beta\) 与前面层参数一并更新。

### 实践建议

1. **对比实验：无 BN vs 有 BN**  
   在简单 CNN（如 MNIST/CIFAR-10）上，固定学习率，对比 train/val loss 曲线与达到目标精度所需 epoch 数。

2. **观察 `train()` 与 `eval()`**  
   ```python
   model.train()   # BN 用 batch 统计，更新 running stats
   model.eval()    # BN 用 running mean/var，行为固定
   ```
   同一输入在两种模式下输出应一致（Dropout 关闭时）；训练模式下 batch 大小变化会影响 BN 层输出。

3. **可视化 batch 统计与 running 统计**  
   注册 hook 打印某 BN 层的 `running_mean`、`running_var` 随 epoch 的变化，理解「历年汇尺」如何形成。

4. **小 batch 实验**  
   将 batch size 从 128 降到 2，观察 BN 性能下降；可进一步了解 Group Norm / Layer Norm。

5. **BN + Dropout 组合**  
   在 MLP 上构建 `Linear → BatchNorm1d → ReLU → Dropout` 栈，与仅 Dropout 对比，体会「稳训练」与「防过拟合」分工。
