# 纹关的抄路司

**日期**：2026-07-24

**对应概念**：残差连接（Skip Connection / Residual Connection）、恒等映射与 ResNet 残差学习

---

## 故事

阿砚在点印司记熟「一格混谱、瓶颈省算」之后，验帛坊又挂新匾：**抄路司**。里正说：「滑印司改形、叠色司增谱、缩格司减格——关越深，帛上的信越要经过越多司吏。今日这一司，专管**另开一条抄近路，把原帛与改过的帛并账相加**。」

抄路司主吏姓并，人称**并公**。并公在案上铺两匹小帛：一匹是**入关原帛** \(X\)，一匹是**深关改过的帛** \(F(X)\)——经滑印、阈门、齐水尺等一串变换后的结果。

「深关要叠很多层，图师却常抱怨：**层数一多，验尺司的差账反而变大**。」并公指着深关册上的一行批注：「不是算力不够，是**信在层层传递里走样、走淡**——像驿路太长，末尾收到的已不是开头那封信的『全文』，只剩残片。」

阿砚问：「多加几司，不就该学得更细吗？」

并公摇头：「学『细』和学『对』不是一回事。若每一司都硬要学**整匹帛从头到尾该怎么变**，层数深了，每一司都要承担『从原样到终样』的全部差事——**任务太重，误差一层层叠上去**，有时还不如浅关。」

---

第二日，并公在深关出口设**并账台**。

流程是这样的：

1. 原帛 \(X\) 从入关处**分作两路**。  
2. **主路**：照常经滑印司、点印司、阈门司等，得到变换后的帛 \(F(X)\)。  
3. **抄路**：不经过那些改形司，**原样抄送**到并账台——若谱数、格数与主路输出一致，这叫**直抄**；若主路里缩格司把八格压成四格、或点印司把四谱合成二谱，抄路司会在关册上配**对齐小吏**：补边、跳格或一格混谱，让抄来的原帛与 \(F(X)\) **形状对齐**，记作 \(\mathcal{A}(X)\)。  
4. **并账**：并公在并账台上写：  
   \[
   Y = F(X) + \mathcal{A}(X)
   \]  
   「**主路负责『改多少』，抄路负责『原来是什么』；合账之后，深关学的往往不是『整匹新帛』，而是『在原帛上要补的那一笔差』。**」

阿砚记：若抄路是**直抄**且不做对齐变换，\(\mathcal{A}\) 就是**恒等**——原帛原格原谱照送，并账式简化为 \(Y = F(X) + X\)。

---

第三事。混公自点印司来，与并公同席议 **瓶颈块**。

深关图师要在主路里排：**点印降谱 → 三格滑印 → 点印升谱**，中间通道窄、算力省；可抄路仍是**入关时的宽谱、大格**。

「宽谱大格与窄路输出的宽谱大格，格数谱数一致，**直抄即可**。」并公在关册画框：主路中间虽窄，**抄路仍从入关处分流，绕开窄路，到出口再与主路汇合**——「窄路只算 \(F(X)\) 里『该改的那一段』；**\(X\) 不必挤过窄路**。」

阿砚悟：这与点印司的「十六谱压四谱再三格印」同阵：主路 bottleneck，抄路**不 bottleneck**。

---

第四事。验尺司的差账问题。

里正拿出两本关册对比：

| 深关结构 | 层数加深时验尺差账 |
|----------|-------------------|
| 纯叠层（只有主路） | 差账先降后**升**，越深越难训 |
| 主路 + 抄路并账 | 差账随层数加深**仍可继续降** |

并公解释：「若网络必须学 \(H(X)\)——直接从 \(X\) 到目标的整映射——深了就难。改成学 \(F(X) = H(X) - X\)，并账 \(Y = F(X) + X\)，**主路只需学『残差』、『增量』**；抄路保证『原信至少能原样到达出口』。最差情况，主路把 \(F\) 学成零，并账仍得 \(X\)——**不会比不加深更差**，这叫**恒等兜底**。」

阿砚问：「主路全学零，不就等于没加深？」

并公：「**可以，但不必**。浅关已经不错的部分，由抄路保住；主路专心学『浅关还学不会的那一点』——**分工之后，深关才叠得起来**。」

---

第五事。关外图师呈 **《关外图说·抄路篇》**，与并路司对照：

- **残差块（Residual Block）**：\(Y = \mathcal{F}(X) + \mathcal{A}(X)\)，常取 \(\mathcal{A}(X)=X\)（**identity shortcut**）。  
- **\(\mathcal{F}\)**：若干卷积、BN、ReLU 等；**\(\mathcal{A}\)**：当 stride 或通道变化时，用 **1×1 卷积 + 跳格** 对齐（点印司、跳格司在抄路上值班）。  
- **ResNet**：BasicBlock（两层 3×3 + 抄路）、Bottleneck（1×1→3×3→1×1 + 抄路）——与点印司、滑印司串联一致。  
- **为何能训很深**：缓解 **梯度消失 / 退化**——反向时，抄路上的梯度可**直传**回入关处，不必全靠主路链式乘积。

旁注：并公在案角写：**ResNet (He et al., 2015)** ——「Deep Residual Learning」；**skip connection** 即抄路，**residual** 即主路学的 \(F(X)\) 那一笔「差账」。

阿砚对照抄路司里 **原帛分流、主路变换、抄路对齐、出口相加、深关可叠、窄路 bottleneck 不堵抄路**——与图说 **Skip Connection / Residual Learning** 同构。

里正问：「此法何名？」

并公在案上写下：

**原帛抄近路，主路只写差；出口一并账，深关叠不累。**

阿砚默念数遍，忽然接上点印司的「瓶颈」与叠嶂关传灯阵的「层数」，脱口而出：

**残差连接**。

---

## 概念解释

### 基础：残差连接是什么？

**残差连接**（Skip Connection / Residual Connection）是深度网络里的一种**捷径结构**：把某一层的输入 \(X\)，与经过若干变换层后的输出 \(F(X)\) **相加**，得到：

\[
Y = F(X) + X
\]

（当通道数或空间尺寸变化时，\(X\) 常经 **1×1 卷积** 或 **stride** 做对齐，记作 \(\mathcal{A}(X)\)，即 \(Y = F(X) + \mathcal{A}(X)\)。）

**直觉**：

- 没有残差时，堆叠层要学的是 **从 \(X\) 到目标特征的完整映射** \(H(X)\)。  
- 有残差时，堆叠层只学 **增量 / 残差** \(F(X) = H(X) - X\)，输出由 **\(F(X) + X\)** 还原。  
- **抄路**（skip）把输入直接送到输出端相加，提供 **恒等映射（identity mapping）** 的通道；若 \(F(X) \approx 0\)，块近似「什么都不改」，网络至少不会比浅层更差——这缓解了 **网络加深反而效果变差** 的 **退化（degradation）** 问题。

**PyTorch 典型写法（BasicBlock 思想）**：

```python
def forward(self, x):
    out = self.conv_block(x)  # F(x)
    out = out + x             # skip: identity
    return self.relu(out)
```

### 进阶：为什么残差有效？

**1. 优化更容易：学「差」比学「整」简单**

若理想映射接近恒等（\(H(X) \approx X\)），普通层要把权重学成「复制输入」；残差块只需把 \(F(X)\) 推向 **0**，往往更快、更稳。

**2. 缓解梯度消失**

反向传播时，损失对 \(X\) 的梯度含 **\(\frac{\partial Y}{\partial X} = \frac{\partial F(X)}{\partial X} + I\)** 中的 **\(+I\)** 项——抄路上梯度可 **不经主路连乘衰减** 而直达前层，深层网络更易训练。

**3. 与 Bottleneck 配合（ResNet-50 等）**

主路：`1×1 降维 → 3×3 空间卷积 → 1×1 升维`；抄路：当输入输出通道、尺寸一致时 **恒等相加**。见 [2026-07-23 点印司](./2026-07-23-纹关的点印司-1x1卷积.md)。

**4. 对齐变换 \(\mathcal{A}(X)\)**

当主路有 **stride=2** 降采样或 **通道翻倍** 时，\(X\) 与 \(F(X)\) 形状不同，不能直接相加。常用 **`Conv2d(..., kernel_size=1, stride=2)`** 同时改通道与尺寸——故事里的「抄路对齐小吏」。

**常见误区**

- 「残差就是多连一根线」——关键是 **相加（element-wise add）**，不是简单 concat；concat 是另一族结构（如 DenseNet）。  
- 「抄路不参与学习」——\(X\) 固定参与前向；**\(\mathcal{A}\)** 里的 1×1 卷积 **有可训参数**；梯度也会回传到抄路分支。  
- 「层越多越好，全靠残差」——残差 **使加深可行**，仍受数据、正则、任务限制；不是无限叠层魔法。  
- 混淆 **残差块** 与 **整个 ResNet**——残差块是单元；ResNet 由许多块 + 首尾卷积 + 全局池化 + 全连接组成。

### 深入：与已学概念的关系

1. **[1×1 卷积](./2026-07-23-纹关的点印司-1x1卷积.md)**  
   Bottleneck 主路与抄路对齐时的 **1×1 + stride** 即点印司、跳格司在抄路上值班。

2. **[卷积、步长、通道](./2026-07-13-纹关的滑印司-卷积.md)**  
   主路 \(\mathcal{F}\) 仍是常规卷积堆叠；残差是 **块级** 拓扑，不改变单层卷积定义。

3. **[反向传播](./2026-06-27-七坊连环的悔信-反向传播.md)**  
   相加节点的梯度 **分给两路**；抄路解释「梯度高速公路」的直观来源。

4. **[批归一化](./2026-07-11-平渠关的齐水尺-批归一化.md)**  
   ResNet 块内 Conv 后常接 BN、ReLU；**相加后再 ReLU**（BasicBlock）是常见顺序。

5. **后续可衔接**  
   - **DenseNet**：改成 concat 而非 add。  
   - **Highway / Residual 变体**：带门的抄路。  
   - **Transformer**：也有 skip + LayerNorm（不同归一化，同「抄路」思想）。

### 实践建议

1. **打印残差块形状**  
   ```python
   import torch
   import torch.nn as nn

   class TinyResBlock(nn.Module):
       def __init__(self, c):
           super().__init__()
           self.f = nn.Sequential(
               nn.Conv2d(c, c, 3, padding=1),
               nn.ReLU(),
               nn.Conv2d(c, c, 3, padding=1),
           )
       def forward(self, x):
           return torch.relu(self.f(x) + x)  # skip

   x = torch.randn(2, 64, 32, 32)
   y = TinyResBlock(64)(x)
   assert y.shape == x.shape
   ```

2. **对比有无 skip 的训练曲线**  
   同样深度（如 20 层小 CNN），一组块内 `out = conv(x)`，一组 `out = conv(x) + x`，在 CIFAR-10 子集上比较 train/val loss——体会退化与残差的差别。

3. **读 Bottleneck 参数量**  
   对照 [点印司](./2026-07-23-纹关的点印司-1x1卷积.md) 手算 `Bottleneck(256→64→256)` 与 `BasicBlock(256)` 的 params/FLOPs。

4. **画数据流图**  
   纸笔画：\(X\) 分叉 → 上路 \(\mathcal{F}\) → 与下路 \(X\)（或 \(\mathcal{A}(X)\)）在 ⊕ 处相加 → ReLU。
