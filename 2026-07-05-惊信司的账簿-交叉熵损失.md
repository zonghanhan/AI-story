# 惊信司的账簿

**日期**：2026-07-05

**对应概念**：交叉熵损失（Cross-Entropy Loss）

---

## 故事

验真镇扼守九条驿道，每日有信使自镇外而来，须从九道门之一入镇投递。镇中设**配票司**（此镇与百票镇同制），据九门的**适宜度原始分**，经指数台归一，把**恰好一百张通行票**分给九门——票额即各门当日的**成数**，九门相加恒为百。

配票司只管「各门占几成」，不管「哪门才是正解」。正解由镇志记载：当日实际应走之门，称为**真门**。真门每日一换，由驿馆根据路况、时令、货种而定；信使若走错门，绕远半日，货损三成。

镇中又设一司，与配票司比邻，名**惊信司**。惊信司不收票、不开门，只收**成数账**——配票司把九门成数抄送一份，惊信司据**真门**记一笔**惊数**。

惊信司的老司正姓陆，人称**陆惊**。陆惊上任时，里正只交代一句：「成数账每日一送，惊数账每日一结。惊数越小，司正越称职。」

---

新任书吏沈括（与百票镇同名，然非一人）第一日收账，见九门成数如下——

| 门号 | 成数 |
|------|------|
| 东门 | 8 |
| 南门 | 12 |
| 西门 | 5 |
| 北门 | 58 |
| 中门 | 7 |
| 水门 | 3 |
| 山门 | 4 |
| 驿门 | 2 |
| 货门 | 1 |

镇志载：**真门为北门**。

沈括的第一反应，与多数新手一样：看**成数差**。北门五十八，东门八，差五十——「差得大，惊数该大。」他提笔记：惊数 = 五十八减八 = **五十**。

陆惊摇头：「北门是真门，你该看的不是北门与东门之差，而是**真门得了几成**。」

沈括改记：惊数 = 百减五十八 = **四十二**——「真门占五十八成，尚有四十二成信错在别处，故惊数为四十二。」

陆惊仍摇头：「若真门占九十九成，按你的算法，惊数仅一——几乎完美。若真门占五十一成，惊数四十九——按你的算法，几乎糟透。可五十一成与五十九成，在验真镇里都是『略偏真门』，不该一个天一个地。」

沈括困惑：「那惊数究竟怎么算？」

陆惊引他到司后一间无窗小室。室中案上置**对数尺**——不是寻常直尺，而是一根刻着弯曲线的木尺，左端标「百成」，右端标「一成」；尺上数字越小，曲线爬升越陡。

---

陆惊令沈括把**真门成数**在对数尺上定位。

「真门五十八成，」陆惊道，「在对数尺上读数，得**负惊初值**约 0.54。惊数账上记作 **0.54**——不是五十，不是四十二，是 **0.54**。」

沈括问：「为何用对数尺？为何成数越高，惊数反而越小？」

陆惊答：「惊信司量的不是『票差多少』，而是**若按你的成数押注，真门发生时的意外程度**。真门占五十八成，意味着你押了五成八的把握在真门上——意外不大，惊数 modest。真门若只占一成，意味着你把九成九的把握押在别处——真门发生时，你**极度意外**，对数尺读数极大，惊数可至 **4.6** 以上。」

沈括试算几组：

| 真门成数 | 惊数（约） |
|----------|------------|
| 90 | 0.11 |
| 58 | 0.54 |
| 30 | 1.20 |
| 10 | 2.30 |
| 1  | 4.61 |

他倒吸凉气：「成数从十降到一，惊数不是多 2.3，而是**翻了一倍还多**——曲线在低端极陡。」

陆惊点头：「这叫**对数惩罚**。略低估真门，惊数温和增加；**自信地错门**，惊数暴涨。验真镇宁可你『 modest 地对』，不要你『 loudly 地错』。」

---

第二日，来了一位**方差派**的辩客，主张：「惊数何必用对数尺？用**平方差**即可——真门成数与百成之差，平方后相加。」

沈括试之：真门五十八成，平方差 = \((100-58)^2 = 1764\)；真门九十成，平方差 = 100。数值动辄千百，且九十成与五十八成的「好」程度，在平方差里不成比例。

陆惊道：「平方差量的是**距离**，对数尺量的是**意外**。成数本质是**比例**，比例的世界，该用**对数**说话——这与百票镇指数台同源：成数来自指数放大再归一，惊数则取**成数之对数的负值**。」

沈括于案头写下陆惊口述的口诀：

> **惊数 = 负的对数（真门成数 ÷ 百）**

真门成数五十八，即 0.58，负对数约 0.54——与对数尺读数一致。

---

第三日，镇志载真门为**东门**，而配票司成数为：北门五十八，东门八，其余零碎。

沈括心算：惊数 = \(-\log(0.08) \approx 2.53\)——比第一日真门五十八成时大得多。

里正验账，叹道：「配票司把票押在北门，真门却在东门——**成数集中错门**，比『成数分散、真门居中』更糟。惊信司的账，就是要逼配票司**把成数往真门上堆**。」

沈括问：「若真门成数为零呢？」

陆惊面色一沉：「对数尺在零处**无读数**——惊数为**无穷**。验真镇规矩：成数最小为一，不得为零。这与配票司指数台『吐数必正』一脉相承。」

---

又过数日，沈括随陆惊赴邻镇**百票镇**交流。百票镇配票司周翁问：「惊信司与配票司，谁先谁后？」

陆惊答：「**同轮同训**。配票司据了望台原始分出成数；惊信司据真门记惊数。训练一日：配票司调整原始分权重，使**惊数下降**——不是让北门成数盲目变大，而是让**真门成数变大**。」

周翁补充：「了望台原始分经指数台归一，正是 Softmax；惊信司取真门成数之负对数，正是**交叉熵**里的那一项。二司合起来，叫 **Softmax + 交叉熵**——百票镇管『怎么分成数』，惊信司管『成数离真门有多远』。」

沈括至此恍然，却尚未点破名目。

---

一日，镇外来挑战者，是**硬配派**余党，主张：「真门既唯一，何必成数？九门中**原始分最高者**标为预测，对则惊数零，错则惊数一——简单。」

沈括试：真门北门，原始分最高者亦是北门——惊数零，完美。  
再试：真门东门，原始分最高者北门——惊数一，与真门成数仅八、惊数 2.53 的案子，硬配一律记「一」，**不分错得有多离谱**。

陆惊道：「硬配只问对错，不问**错有多自信**。惊信司要的是：你错在 ninety-nine 成押北门、真门东门时，比 five 成押北门、真门北门时，**罚得更重**——梯度才能告诉配票司『往哪推』。」

沈括又问：「若九门中两门真需比较——比如只有『是』与『否』两道门，成数各五十，真门为『是』，惊数多少？」

陆惊：「两门的账，惊信司只记**真门那一项**——\(-\log(0.5) \approx 0.69\)。假门成数不必单独入账，因二者之和为百，知真门成数即知假门。」

---

训练到第七旬，沈括已能闭卷报惊数。一日里正设考：「真门货门，成数：货门 45，其余九门各约 6——」

沈括：「惊数约 0.80。」

里正：「真门货门，成数：货门 2，北门 91——」

沈括：「惊数约 3.91。后者虽『猜中货门有票』，却把九成一成数押错，**远比前者糟**。」

里正点头，此时方道：「沈括，你入司以来，记的每一笔惊数，在算学上名**交叉熵损失**——Cross-Entropy Loss。Softmax 把原始分变**成数（概率）**；交叉熵用**负对数**量『预测成数与真门（One-Hot 真标签）差多远』。你第一日的 fifty、四十二，是凡人的**线性直觉**；陆惊的对数尺，才是机器学习里**分类任务的标准罚则**。」

沈括回望司中账簿，账页上每日一行：

```
惊数 = -log(p_真门)
```

百页账，百次真门；求其均，曰**平均交叉熵**——验真镇评判配票司优劣的**唯一主账**。

陆惊补最后一句：「**最大似然估计**，在分类里，就是最小化交叉熵。你最小化惊数，等价于最大化『真门成数』的连乘概率——对数把乘变加，负号把最大变最小。惊信司与配票司，合起来，就是在做**现代神经网络分类最后一层**该做的那件事。」

沈括合账，长揖：「学生今日方知，惊数非惊人之数，而是**信使对真门所感到的惊**——成数越偏真门，惊越小；越信错门，惊越大。此理，对九门如此，对万类标签亦然。」

## 概念解释

### 1. 交叉熵损失是什么？

**交叉熵（Cross-Entropy）** 用来衡量两个概率分布之间的差异。在分类任务中：

- **预测分布** \(\hat{p}\)：模型输出（通常经 Softmax 后的各类概率）
- **真实分布** \(p\)：One-Hot 标签（真类为 1，其余为 0）

对单个样本，**多类交叉熵损失**为：

\[
L = -\sum_{k=1}^{K} p_k \log \hat{p}_k
\]

因 One-Hot 只有真类 \(y\) 为 1，其余为 0，上式简化为：

\[
L = -\log \hat{p}_y
\]

即故事中「**惊数 = -log(真门成数)**」——只关心**模型给真类分配的概率**。

### 2. 为何用负对数，而非「百减成数」？

| 真类概率 \(\hat{p}_y\) | \(-\log \hat{p}_y\) | 直觉 |
|------------------------|---------------------|------|
| 0.90 | 0.11 | 很有把握，小罚 |
| 0.58 | 0.54 | 中等把握 |
| 0.10 | 2.30 | 严重低估真类 |
| 0.01 | 4.61 | 几乎押错，重罚 |

**对数惩罚**的特点：

1. **正确且自信** → 损失接近 0
2. **正确但不自信** → 适度损失
3. **错误且自信** → 损失极大（梯度也大，利于学习）

「百减成数」是线性的，无法区分「51% vs 59%」这类接近正确的情形，也无法对「99% 押错类」施加足够惩罚。

### 3. 与 Softmax 的关系

上一讲百票镇的 **Softmax** 把原始 logits 变成概率向量 \(\hat{p}\)。  
本讲惊信司的 **交叉熵** 用真标签 \(y\) 评价这个 \(\hat{p}\)。

二者常**联合使用**，构成分类模型最后一层的标准组合：

```
logits → Softmax → 概率 → Cross-Entropy(与真标签)
```

PyTorch 的 `nn.CrossEntropyLoss` 内部包含 `LogSoftmax + NLLLoss`，输入为 **logits**（未 Softmax），数值更稳定——相当于「指数台与对数尺合一」。

### 4. 二分类：二元交叉熵

只有「是 / 否」两类时，公式为：

\[
L = -\big[ y \log \hat{p} + (1-y) \log (1-\hat{p}) \big]
\]

\(y \in \{0,1\}\) 为真标签，\(\hat{p}\) 为「是正类」的预测概率。故事中「两门只记真门一项，因二者成数之和为百」即此特例。

### 5. 与最大似然估计（MLE）

若把 \(\hat{p}_y\) 看作模型参数下观测到真类的**似然**，最大化 \(\prod_i \hat{p}_{y^{(i)}}\) 等价于最小化 \(-\sum_i \log \hat{p}_{y^{(i)}}\)——即**最小化交叉熵 = 最大似然**。陆惊说的「最小化惊数 = 最大化真门成数连乘」，正是 MLE 的对数形式。

### 6. 为何分类不用均方误差（MSE）？

对 Softmax 输出用 MSE 梯度常**饱和缓慢**，错误时惩罚不够「尖」，训练效率低。交叉熵与 Softmax 组合时梯度形式简洁（\(\hat{p} - y\)），是分类任务的**标准选择**——故事中「方差派平方差」的类比。

### 7. 批量训练中的平均损失

验真镇「百页账求均」对应：

\[
L_{\text{batch}} = \frac{1}{N} \sum_{i=1}^{N} -\log \hat{p}_{y^{(i)}}
\]

\(N\) 为 batch 大小；优化器（如 SGD、Adam）最小化该均值，更新配票司（模型）参数。

### 8. 常见变体（延伸）

- **Label Smoothing**：真标签不全为 1，略分概率给其他类，防止过度自信（对应「真门成数不必逼到 99.9%」）
- **Focal Loss**：对易分样本降低权重，专注难分样本
- **类别权重**：不平衡数据集中对少数类加大惊数权重

### 代码实践

7. **NumPy 手写交叉熵**  
   ```python
   import numpy as np
   def softmax(z):
       z = z - z.max()
       e = np.exp(z)
       return e / e.sum()
   logits = np.array([2.1, 0.3, -1.0, 4.5, 1.2, -0.5, 0.8, 0.1, -0.2])
   probs = softmax(logits)
   true_class = 3  # 北门（0-indexed）
   loss = -np.log(probs[true_class])
   print(f"真类概率 {probs[true_class]:.4f}, 交叉熵 {loss:.4f}")
   ```

8. **PyTorch `nn.CrossEntropyLoss`**  
   - 文档：https://pytorch.org/docs/stable/generated/torch.nn.CrossEntropyLoss.html  
   - 输入：**logits** `(N, C)` 与 **类别索引** `(N,)`（非 One-Hot）；勿先 Softmax。

9. **scikit-learn `log_loss`**  
   - 文档：https://scikit-learn.org/stable/modules/generated/sklearn.metrics.log_loss.html  
   - 用于评估已训练模型的平均交叉熵；可与 `predict_proba` 配合。

### 进阶延伸

10. **信息论视角：熵、交叉熵、KL 散度**  
    - 交叉熵 \(H(p,\hat{p}) = H(p) + D_{KL}(p\|\hat{p})\)；One-Hot 下 \(H(p)=0\)，最小化交叉熵 ≈ 最小化 KL。  
    - 推荐：Cover & Thomas《Elements of Information Theory》第 2 章（本科信息论选修）。

11. **Label Smoothing（Szegedy et al., 2016）**  
    - 论文：*Rethinking the Inception Architecture* — 软化 One-Hot，改善泛化与校准。

12. **Focal Loss（Lin et al., 2017）**  
    - 论文：*Focal Loss for Dense Object Detection* — 类别不平衡与难易样本权重。

### 实践建议

- 固定一组 logits，手算 Softmax 与 \(-\log p_y\)；再改真类索引，观察惊数变化。  
- 对比 **MSE vs Cross-Entropy** 在同一分类任务上的训练曲线（MNIST 一层网络即可）。  
- 画 **损失曲线**：横轴 epoch，纵轴 train/val 交叉熵——结合已学「过拟合」观察 val 损失何时回升。  
- 结合 **Softmax 温度**：同一 logits 用 \(T=0.5, 1, 2\) 算交叉熵，理解「成数更尖/更平」如何影响损失与梯度。
