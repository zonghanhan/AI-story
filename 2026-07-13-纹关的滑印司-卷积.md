# 纹关的滑印司

**日期**：2026-07-13

**对应概念**：卷积（Convolution）与卷积神经网络基础

---

## 故事

青岚山下行，过了平渠关，又逢一关，唤作**纹关**。关外行人须出示**纹帛**——一方方织就的图样，上有深浅不一的经纬，或绘关隘、或绘舟车、或绘禽兽。关内传灯阵虽已训得颇精，阿砚持矩司、齐水尺诸法至此，里正却另出一题：

「阵若把整张纹帛**逐点逐线**读入，一张百尺之帛，便要有百尺之平方的连线——线太多，训不动，也记不牢。且帛上**相邻之纹**本是一体，远处之纹与近处往往无关；逐点全连，既费算力，又乱章法。你须另设一司，教阵**先看局部，再合全局**。」

阿砚初不解，里正引他至关内**验帛坊**。

---

坊中悬一幅巨帛，长宽各三百格，每格染深浅不一的色，合起来像一幅山水。老吏道：「若令传灯阵**每一格**都与**每一根灯柱**相连——你叠嶂关里唤作『全连』——则入阵之线数以格数平方计，三十万格便有**近亿条细线**。训一张帛，矩司合降要调亿计之权；换一张稍大的帛，格数一变，线数又变，阵便**无法复用**。」

阿砚问：「那该如何？」

老吏不答，只引他见**滑印司**。

---

滑印司主吏姓印，人称**印公**。印公案上不见巨簿，只有几枚**小印**——最大不过三格见方，最小仅二格见方。每枚小印上刻不同**纹样**：一枚左深右浅，像一道**竖边**；一枚上深下浅，像一道**横边**；一枚中心一点四周浅，像**小圆**；一枚斜纹交错，像**折角**。

印公道：「验帛之法，不在一次读全幅，而在**持印滑行**。」

他取「竖边印」，按在巨帛**左上角**三格见方之处，使印中每一格与帛上对应之格**逐格相叠**——印中深者，遇帛上亦深，则**相乘后相加**得**大数**；印中深、帛上浅，则得**小数**。印公将这一叠九格之**合分**记在旁侧一张小帛的**同一位置**，唤作**响应一格**。

随后，印公将竖边印**向右挪一格**，再叠再算，再记响应；再挪，再算——直至巨帛**最右列**滑完；再**下移一行**，从左侧重滑。整幅巨帛滑毕，小帛上便浮出一幅**新的图**：原帛上凡**竖边明显**之处，响应格**亮**；无竖边之处，响应格**暗**。

阿砚问：「为何不用巨印一次盖全幅？」

印公道：「巨印要识**何种**纹？竖边？横边？圆点？纹在帛上**何处**出现？小印只问一件事——**这一小块里，有没有我刻的这种纹？** 滑过全幅，便答『在何处、有多强』。此谓**局部探纹**。」

---

阿砚又问：「一枚竖边印滑完，只得一种响应图。可巨帛上还有横边、圆点、折角……」

印公指向司内**十二名印匠**，各持一印，同时从**同一左上角**起滑。十二人滑法相同——**同格数、同步长、同步幅**——所刻纹样却不同。滑毕，案上十二张小帛，称**十二通道**：

- 竖边印 → 竖边在何处强；
- 横边印 → 横边在何处强；
- 圆点印 → 圆点在何处强；
- ……

阿砚恍然：「不是一根灯柱读全帛，而是**多枚小印各读一遍局部**，各得一张**响应图**。」

印公点头：「且同一枚竖边印，滑过左上与滑过右下，**印上纹样不变**——印中九格之权，**全幅共用**。此谓**同印同权**；若逐格各设一套权，又是亿计之线。同印同权，参数量便从『格数平方级』降为『印格数级』。」

阿砚默记：**小印**、**滑行**、**逐格相叠求合**、**响应图**、**多印多通道**、**同印同权**。

---

里正来验，命阿砚将此法接入传灯阵。

阿砚起初仍用**全连**：巨帛展平成一长列，送入叠嶂关式灯柱——训至第三日，矩司合降震荡，内存亦告急。里正摇头：「全连把**相邻**与**相隔千里**的格等而视之，帛左上角一块水纹，与右下角一块云纹，本不该用同一套线硬连。」

阿砚改设**滑印层**：

1. 巨帛**不展平**，保留**行×列**之二维；
2. 每层设若干枚**可训之印**（初值随机，非印公那几枚固定的）；
3. 每枚印在输入帛上**滑一遍**，得一张响应图；
4. 多张响应图叠成**本层输出**，再经阈门司（激活）、齐水尺（批归一化），送下一层。

第二旬，阵已能辨帛上**简单纹**：竖边、横边、小圆。阿砚发现：**第一层**的响应图，像印公案上那十二张小帛——亮处是「这一局部像不像某种边或点」；**第二层**再设滑印，在**响应图**上继续滑——像在「已标出竖边之处」附近再探「是否形成框」「是否连成线」。越往后，小印所探之块虽仍只有几格见方，因前层已把**低级纹**标亮，后层组合的是**更高级的形**。

里正问：「何以第二层能识**框**与**线**，而非仅识**边**？」

印公道：「**边**是第一层小印在**原帛**上滑出的响应；**框**是第二层小印在**第一层响应图**上滑出——它看的已不是原始经纬，而是『竖边在左、横边在上、此处皆亮』这类**组合**。层数叠上去，**局部探纹**逐层堆高，终识**舟车禽兽**。」

---

关内亦有细规，印公一一示之。

**步幅。** 小印每次挪几格？挪一格，响应图与帛**同大**，称**步幅为一**；挪两格，响应图**缩小**，算得少、滑得快，但可能**漏纹**。阿砚在快验与细验之间取**步幅一**训阵，**步幅二**做粗筛，里正允之。

**边距。** 帛边上一圈格，小印若只从左上角起滑，**最右、最下**几列是否也要探？印公在帛边**补一圈浅格**（或令小印**越界处按零计**），使响应图**不缩边**——唤作**补边**。

**叠印。** 若两枚小印的**印格数**不同——一枚三格见方、一枚五格见方——同层并用，则各滑各的，得不同大小的响应图；再经**缩格司**（阿砚后来才知，唤作**池化**）对齐，或分置不同层。印公道：「同层之印，**印格数宜同**；**印格数**本身可训、可设，是为**感受野**之大小。」

阿砚问：「感受野？」

印公道：「小印一次能盖住的格数，便是此层『看得有多宽』。层数越深、印越大，阵在帛上『一眼』所及越广——但**每一眼仍只看局部**，只是局部里已含前层标好的**抽象纹**。」

---

第三旬，阿砚几乎训毕，里正命他**封关实守**——关外行人持纹帛而来，阵须实时判是「关牒」「路引」还是「伪帛」。

封关那日，阿砚立于阵前，见一行人呈上一帛。阵内第一层十二枚印同时滑过，毫秒间十二通道响应已出；第二层、第三层依次滑印、阈门、齐水；末层展平，接百票镇配票司与惊信司账簿——输出三类概率。阿砚忽生一惑：「若整张帛展平全连，末层灯柱数需与格数平方相配；如今**滑印层**之后，末层所接之**展平长度**，仅为**最后一层响应图之格数×通道数**，远小于原帛。」

印公取出一册**《关外图说》**，翻到一页，上画：

- 左：巨帛，上覆一小方框，框下写「**核**（kernel）滑过**输入**」；
- 中：小帛响应图，写「**特征图**（feature map）」；
- 右：多层小框相叠，写「**卷积层** → **激活** → **再卷积**」。

阿砚凝视「核」字，又看司中匠人**持印滑帛**之姿——那小印每次挪一格、逐格相叠、全幅共用同一印纹——与图说中**核在输入上滑动、做点积、共享权重**完全同构。

里正问：「此法何名？」

印公在案上写下二字：

**卷积**。

阿砚起初以为，纹关不过是设一司，用小印在帛上滑来滑去，看局部有没有某种纹，免得全幅逐点全连。直到见《关外图说》与图说中 **ConvNet** 字样，他才忽然明白：滑印司所教阵的，并不只是「省几条线」，而是深度学习里处理**图像、网格、空间结构**数据的核心算子——**卷积**；而多层滑印叠成的传灯阵，便是**卷积神经网络（CNN）** 的骨架。

## 概念解释

### 基础：卷积是什么？

**卷积（Convolution）** 是卷积神经网络（CNN）的核心运算。其思想与故事中的「滑印」一致：

> 用一个小矩阵（**卷积核 / 滤波器，filter / kernel**）在输入数据（如图像）上**滑动**，在每个位置计算核与对应局部区域的**逐元素乘积之和**，得到输出矩阵（**特征图，feature map**）。

对二维图像，一个 \(3 \times 3\) 的核在 \(H \times W\) 的输入上滑动，每个输出位置：

\[
y[i,j] = \sum_{m}\sum_{n} x[i+m, j+n] \cdot w[m,n] + b
\]

故事与概念的对应：

| 故事元素 | 机器学习概念 |
|----------|--------------|
| 巨帛 / 经纬格 | 输入图像（像素网格） |
| 小印 / 印上纹样 | 卷积核（kernel / filter）的权重 |
| 持印滑过、逐格相叠求合 | 卷积运算（滑动窗口 + 点积） |
| 响应小帛 | 特征图（feature map） |
| 十二印匠、十二通道 | 多个卷积核 → 多个输出通道 |
| 同印同权、全幅共用 | **参数共享**（同一核在整个输入上复用） |
| 局部探纹 | **局部连接**（每个输出只看一小块输入） |
| 步幅 | **步长（stride）** |
| 补边 | **填充（padding）** |
| 多层滑印叠用 | 多层卷积堆叠 |
| 第二层在响应图上滑 | 深层网络在**前层特征图**上继续卷积 |
| 滑印层 + 传灯阵 | **卷积神经网络（CNN）** |

**为何需要卷积？** 全连接层把每个像素与每个神经元相连，参数量 \(O(\text{像素数} \times \text{神经元数})\)，且**忽略空间结构**——相邻像素与 distant 像素被同等对待。卷积通过**局部感受野**和**权重共享**：

1. **大幅减少参数**（同一核在全图复用）；
2. **利用平移不变性**——猫在左上角与右下角的「边缘」用同一组核检测；
3. **保留空间结构**——输出仍是二维网格，便于堆叠多层、逐层抽象。

**直觉**：第一层卷积检测**低级特征**（边、角、色块）；后续层在特征图上再卷积，组合成**中级特征**（纹理、部件）和**高级特征**（物体、类别）——对应「先看边，再看框，再看舟车禽兽」。

### 进阶：CNN 的基本组件

1. **卷积层（Conv Layer）**  
   - 超参数：**核大小**（如 \(3\times3, 5\times5\)）、**步长**、**填充**、**输出通道数**（核的个数）。  
   - 每个核产生一张特征图；\(C_{\text{out}}\) 个核 → \(C_{\text{out}}\) 张特征图。

2. **激活函数**  
   卷积后通常接 ReLU 等非线性激活（参见 [2026-06-30 石泉巷的阈门司 — 激活函数](./2026-06-30-石泉巷的阈门司-激活函数.md)），否则多层卷积等价于单层线性变换。

3. **池化层（Pooling）**  
   对特征图**降采样**（如 \(2\times2\) 窗口取最大值 → Max Pooling），缩小尺寸、减少计算、增加感受野，并带来轻微平移鲁棒性。故事中的「缩格司」即池化；本系列后续可专篇展开。

4. **全连接层（FC）**  
   若干卷积 + 池化后，特征图**展平**，接全连接层做分类（配合 Softmax、交叉熵，参见 [2026-07-04](./2026-07-04-百票镇的配票司-Softmax与多分类.md)、[2026-07-05](./2026-07-05-惊信司的账簿-交叉熵损失.md)）。

5. **典型结构**  
   `[Conv → BN → ReLU → Pool] × N → Flatten → FC → Softmax`  
   现代实现常把 BN 放在 Conv 与 ReLU 之间（参见 [2026-07-11 平渠关的齐水尺 — Batch Normalization](./2026-07-11-平渠关的齐水尺-批归一化.md)）。

6. **输出尺寸（无 padding、stride=1）**  
   输入 \(H \times W\)，核 \(K \times K\)，则输出边长约为 \(H-K+1\)。**padding** 可在边缘补零以保持尺寸；**stride>1** 会缩小输出。

7. **感受野（Receptive Field）**  
   网络中某一层的一个神经元，在**原始输入**上能「看到」的区域大小。层数越深、核越大、池化越多，感受野越大——对应「每一眼仍只看局部，但局部里已含抽象纹」。

### 深入：与已学概念的关系

1. **vs 全连接神经网络**（[2026-06-29 叠嶂关的传灯阵](./2026-06-29-叠嶂关的传灯阵-神经网络.md)）  
   CNN 可看作带**结构先验**的网络：局部连接 + 共享权重。全连接是 CNN 的特例（核与输入同大、不滑动），但几乎从不用于大图。

2. **训练方式不变**  
   卷积层权重仍通过**反向传播**（[2026-06-27 七坊连环的悔信](./2026-06-27-七坊连环的悔信-反向传播.md)）与**梯度下降 / Adam**（[2026-06-25 雾谷里的拾阶人](./2026-06-25-雾谷里的拾阶人-梯度下降.md)、[2026-07-09 分径巷的矩司](./2026-07-09-分径巷的矩司-Adam优化器.md)）更新；**损失函数**（[2026-06-28 验尺司的差账簿](./2026-06-28-验尺司的差账簿-损失函数.md)）与分类任务相同。

3. **正则与泛化**  
   Dropout、权重衰减、数据增强常用于 CNN；**过拟合**（[2026-06-26 藏书阁里的抄写生](./2026-06-26-藏书阁里的抄写生-过拟合与泛化.md)）在图像任务中尤为常见（参数仍可能很多）。

4. **应用领域**  
   图像分类、目标检测、语义分割、人脸识别、医学影像等；一维卷积用于序列，三维卷积用于视频/体数据。

5. **历史里程碑**  
   LeNet（1998）→ AlexNet（2012）→ VGG / GoogLeNet / ResNet 等，推动现代计算机视觉；**ResNet 残差连接**、**注意力**（[2026-06-25 灯塔镇的回声集市 — 注意力机制](./AI寓言故事.md)）等是在 CNN 之上的进一步演进。

### 实践建议

1. **手算小例子**  
   用 \(3\times3\) 输入、\(2\times2\) 核、stride=1、无 padding，手算输出特征图每个位置的值，再与 `scipy.signal.convolve2d` 或 PyTorch 对照。

2. **MNIST / CIFAR-10 小 CNN**  
   ```python
   nn.Sequential(
       nn.Conv2d(1, 32, 3, padding=1), nn.ReLU(),
       nn.MaxPool2d(2),
       nn.Conv2d(32, 64, 3, padding=1), nn.ReLU(),
       nn.MaxPool2d(2),
       nn.Flatten(), nn.Linear(64*7*7, 10)
   )
   ```
   打印每层输出 `shape`，理解 `H, W, C` 如何变化。

3. **可视化卷积核与特征图**
