# 缓坡关的步幅吏

**日期**：2026-07-07

**对应概念**：学习率（Learning Rate）

---

## 故事

青岚山西麓有处**缓坡关**，关下是百里雾谷的入口。关前立着一块**等高碑**——碑面刻满细密等高线，最低处唤作**谷底**，是采药人、行商、学徒都要抵达的地方。雾谷里本无台阶，只有一条看不见尽头的斜面；要下到谷底，全凭关里发的**步幅尺**——尺上每一格，代表下山时该跨多大一步。

这步幅尺，与筹巷里秤的步长绳、雾谷里阿禾拾阶的规矩，本是同一脉。

---

缓坡关有三位步幅吏，各守一亭，各发一尺，争了多年。

**急跨派**的吏正姓韩，人称**韩阔**。韩阔的步幅尺极宽：每跨一格，等于常人三步。他站在关楼上喊：「雾谷深，路远，步子小了，一辈子也到不了谷底！」

起初，过路的学徒**小满**试韩阔的尺，果然快——第一日便从关前下到等高碑的**中段**，比旁人快出数倍。可再往下，怪事来了：小满常一步跨过头，踩进侧边的泥沼，又被弹回稍高的坡面；有时连跨两格，竟比起点还高。草图上轨迹如锯齿，来回震荡，三日过去，人还在中段打转。韩阔仍不松尺：「那是你脚力不够，再加大跨幅！」

**细挪派**的吏正姓温，人称**温寸**。温寸的步幅尺极窄：一格只及常人半步。小满换温寸的尺，再下山，这回不再弹回——每步都稳稳向下，绝无泥沼。可一日只能挪下**两格**，等高碑上密密麻麻，从关前到谷底，温寸估摸要**三百日**。有急事的人等不起，有学徒算过：「按这尺，采药季都过了，人还在坡中段。」

---

关里正久欲定规。直到陆惊——筹巷里试过分批秤的那位——路过缓坡关，见小满在中段扎营，已七日未到谷底。陆惊问：「你二人可曾试过，**不大不小、且随坡变**的步幅？」

韩阔道：「步子小了，何时到底？」

温寸道：「步子大了，便要弹回，不如不动。」

陆惊不辩，只请二人各借一学徒，用同一等高碑、同一起点，试**第三把尺**。

---

陆惊的尺，名**缓降尺**，规矩有三：

**其一，初跨适中。** 第一格，既非韩阔的三步宽，也非温寸的半步窄，恰是常人**一步**——小满试跨，一日下到中段偏下，无泥沼，无弹回。

**其二，越近越小。** 陆惊令小满每下一层等高线，便让书吏在尺上**收半格**——坡愈缓、离谷底愈近，步子愈小。韩阔曾笑：「快到谷底反而缩步，岂不是更慢？」陆惊答：「坡面在此最滑，大步易过头；小步微调，才贴谷底。」小满依此再下，最后几层不再震荡，稳稳落在**谷底**标记上。

**其三，遇沼暂收。** 若某段侧有泥沼、梯度乱，陆惊不令停步，只令**临时收一格**——待过乱坡，再恢复。小满说：「这与筹巷里秤绳时，遇噪声笔记便略减变幅，是同一道理。」

---

试尺第七日，里正召集三亭。韩阔仍不服：「陆惊的尺，初日还不如我快。」

陆惊令书吏取来三人的**等高轨迹图**：横轴是「所跨步数」，纵轴是「当前高度」。

- **韩阔尺**：前段骤降，后段锯齿震荡，**七日未到底**；
- **温寸尺**：曲线平滑，七日仅到**全程三分之一**；
- **缓降尺**：前段稳降，后段略缓，**第七日落谷底**。

陆惊道：「韩阔给的是**过大步幅**——方向虽对，一步跨太远，越过谷底便弹回，永远在山腰来回。温寸给的是**过小步幅**——方向也对，可每步太谨慎，更新太慢，误期误事。缓降尺给的是**合宜步幅**：初时一步，够快且稳；近谷底时收步，防过头；遇乱坡暂收，防偏航。」

里正问：「这一格的大小，可有定数？」

陆惊答：「无万全定数，只有权衡。坡陡、人初下山，格可略宽；坡缓、近目标，格须略窄——这叫**随程收步**。若格宽如韩阔，训练发散；若格窄如温寸，收敛太慢。合宜之格，须与**分批秤**、**降向**一并调——同一降向，步幅不同，结果天差地别。」

里正问：「此法何名？」

陆惊在案上写下四字：

**步幅循降**。

小满起初以为，说的是缓坡关里发尺的规矩。直到数年后他回关整理旧册，见陆惊所留图旁，另有一图：横轴是「迭代次数」，纵轴是「损失值」——一条曲线在步幅过大时上下震荡不收敛，一条在步幅过小时几乎水平，一条在**适中步幅且逐段衰减**时，平滑落至谷底——他才忽然明白，关里称的，并不只是下山。

## 概念解释

### 基础：学习率是什么？

在机器学习中，**学习率（learning rate，常记作 η 或 α）** 是控制**每次参数更新幅度**的超参数。

梯度下降的核心更新公式为：

\[
\theta_{t+1} = \theta_t - \eta \cdot \nabla_\theta L(\theta_t)
\]

其中：
- \(\theta\)：模型参数（故事里的「步幅尺刻度 / 当前位置」）
- \(\nabla_\theta L\)：损失函数对参数的梯度（故事里的「降向 / 该往哪边走」）
- \(\eta\)：**学习率**（故事里的「每一格该跨多大一步」）

**梯度只告诉你方向，学习率决定走多远。** 方向错了要改算法；方向对了但步子不对，同样到不了「谷底」（损失最小点）。

故事里的对应关系：

| 故事元素 | 机器学习概念 |
|----------|--------------|
| 步幅尺 / 每一格 | 学习率 η |
| 降向 / 下山方向 | 梯度 \(\nabla L\) |
| 谷底 | 损失函数最小值（最优参数） |
| 韩阔的宽尺 | 学习率过大 |
| 温寸的窄尺 | 学习率过小 |
| 越近谷底收半格 | 学习率衰减（learning rate decay） |
| 遇沼暂收 | 在困难区域降低学习率 / 自适应调整 |

### 进阶：学习率过大 vs 过小

**学习率过大**（韩阔尺）：
- 参数更新步子太大，可能**越过**最优点；
- 损失曲线剧烈震荡，甚至**发散**（loss 越来越大）；
- 表现为训练不稳定，验证指标忽高忽低。

**学习率过小**（温寸尺）：
- 每步更新太保守，收敛**极慢**；
- 需要极多 epoch 才能到达较优解，训练成本高；
- 在复杂损失曲面上，还可能困在**平坦区域**，进展几乎停滞。

**合适的学习率**（缓降尺）：
- 训练初期可较快下降；
- 接近最优时适当减小，避免在最优附近「来回弹跳」；
- 与 batch size、优化器类型需**联合调节**（筹巷篇已述：batch 大时梯度更准，有时可略增学习率）。

### 深入：学习率调度与常见策略

实际训练中，固定一个学习率往往不够，常用**学习率调度（learning rate scheduling）**：

1. **Step decay（阶梯衰减）**  
   每若干 epoch 将学习率乘以一个小于 1 的因子——对应故事里「每下一层等高线，收半格」。

2. **Cosine annealing（余弦退火）**  
   学习率按余弦曲线从初值平滑降至接近 0，后期微调更细。

3. **Warmup（预热）**  
   训练最初若干步先用较小学习率，再升至目标值——避免初期梯度噪声大时步子过猛（类似过乱坡前先收一格）。

4. **自适应优化器（Adam、RMSprop 等）**  
   为每个参数自动调节有效步长；但**全局基础学习率**仍常需手动设定，且 Adam 的默认 1e-3 并非对所有任务最优。

5. **学习率搜索**  
   - **Learning rate range test**：从小 lr 扫到大 lr，观察 loss 何时下降最快、何时发散；  
   - 常用起点：SGD 用 0.01–0.1，Adam 用 1e-4–1e-3，需视任务与 batch size 调整。

### 与已学概念的联系

- **梯度下降**（2026-06-25）：学习率是梯度下降公式中唯一显式控制「步长」的旋钮。  
- **小批量梯度下降**（2026-07-06）：每个 batch 算出的梯度方向相同，但学习率决定沿该方向更新多少；batch 噪声大时常需更小 lr。  
- **反向传播**（2026-06-27）：反向传播给出 \(\nabla L\)，学习率决定 \(\Delta \theta = -\eta \nabla L\) 的大小。  
- **过拟合与正则化**（2026-06-26、2026-07-01）：学习率过大可能导致训练不稳定；过小可能欠拟合或训练不足——需与 early stopping、正则化配合。  
- **损失函数**（2026-06-28）：学习率影响在损失曲面上「如何走」，不改变损失定义本身。

### 实践建议

1. **固定其他条件，只改学习率**  
   用同一模型与 batch size，分别试 `lr = 1e-1, 1e-2, 1e-3, 1e-4`，绘制 train loss 曲线，观察震荡、发散与收敛速度的差异——对应韩阔尺、温寸尺与缓降尺。

2. **使用 Learning Rate Finder**  
   - PyTorch Lightning 的 `lr_finder` 或 fastai 的 `learn.lr_find()`，快速找到较合适的 lr 区间。

3. **尝试学习率调度**  
   - PyTorch：`StepLR`、`CosineAnnealingLR`、`OneCycleLR`；对比固定 lr 与 decay 后的验证集表现。

4. **记录实验**  
   用 TensorBoard / wandb 同时记录 lr 与 loss，理解「近谷底收步」在曲线上的形态。
