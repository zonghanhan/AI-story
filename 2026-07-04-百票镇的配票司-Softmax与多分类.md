# 百票镇的配票司

**日期**：2026-07-04

**对应概念**：Softmax 函数与多分类（Softmax & Multi-class Classification）

---

## 故事

百票镇扼守七条商道，镇中有一司，专管**开门**——镇外每日有旅人携货而来，须从七道门之一入镇；门Wrong则绕远路，货损半日。故百票镇设**配票司**，每日根据七门的**适宜度**，把**恰好一百张通行票**分给七门——票多者，当日开度大；票少者，开缝窄。

配票司不收银两，只收**评票**：七门各递一张**原始分**，分高者更宜通行。评票由镇外七处**了望台**报送——了望台看云色、路泥、风向、货种，各说各话，分数尺度不一；有的台惯打高分，有的台惯打低分，有的台把差门也打成六十分，有的台把好门也只给七分。

新任司正沈括上任第一日，里正便递来难题：「七门原始分已齐，请配票。」

沈括把七张评票摊在案上——

| 门号 | 原始分 |
|------|--------|
| 东门 | 2.1 |
| 南门 | 0.3 |
| 西门 | -1.0 |
| 北门 | 4.5 |
| 中门 | 1.2 |
| 水门 | -0.5 |
| 山门 | 0.8 |

沈括的第一反应，与多数新手一样：**按分比例直分**。他把七数相加得 7.4，再令每门得票 = 原始分 ÷ 7.4 × 100。算下来北门约 60.8 张，西门约 -13.5 张——**负数**。

吏员哄笑：「司正，票岂能为负？西门若得负票，难道还要向镇里倒贴通行权？」

沈括改法：把负分一律抬成零，再按比例分。西门变零，其余重算——可里正摇头：「西门原始分虽负，却非无意义；它表示『较不宜』，不是『与东门无差』。你把它抬成零，等于说西门与南门一样糟，了望台却明明说西门更糟。」

---

配票司的老书吏姓周，绰号**归一翁**，见沈括为难，引他到司后一间小室。室中无窗，只有一座**指数台**——台上有七格，每格对应一门；原始分投入格中，经台内机括一翻，吐出的不再是原数，而是**正数**，且**数目越大，吐出的增量越猛**。

周翁解释：「原始分可正可负，可大可小；配票却必须为正，且百票之和须为百。指数台做的第一件事，是把分差**拉开**——高分更高，低分更低，负分变正却极小。这叫**指数放大**，不是为了让某门独吞百票，而是为了让『更宜』与『较不宜』在配票上**可比较**。」

沈括试投东门 2.1，指数台吐出 8.2；投西门 -1.0，吐出 0.37；投北门 4.5，吐出 90.0。同一台，分数差一倍，吐出的数差远不止一倍——**小分更小，大分更大**。

周翁道：「第二步，把七格吐出之数**相加**，再令每门得票 = 该门吐出数 ÷ 总和 × 100。如此，票必为正，且七门票额之和**恒等于一百**——这叫**归一**。」

沈括依此法配票：北门得约 58 张，东门约 5 张，西门约 0.3 张……七门相加，厘毫不差，正是百。

里正验过，点头：「这才像话。西门不是零，只是极少——符合了望台『较不宜』之意。」

---

沈括以为万事大吉，谁知第二日来了**硬配派**的辩客。

硬配派主张：「百票何须分？七门中**原始分最高者独得百票**，其余六门得零。开门一事，非开即关，何必把票撕成碎片？」

沈括试之：北门 4.5 最高，百票全给北门。旅人拥至北门，队如长龙；东门其实亦宜，却因 4.4 分之差，一票未得，闲死一日。

周翁引沈括看镇志：**百票镇的货种，每日不同**。有时北门最宜大宗货运，有时东门宜轻便脚夫；硬配只看「谁第一」，不看「次席差多少」。若北门 4.5、东门 4.4，硬配与北门独吞无异；若北门 4.5、东门 1.0，硬配仍让北门独吞——两种情形，硬配结果相同，了望台却明明说第二种里「次席远逊」，第三种里「次席逼近」。

「硬配，」周翁道，「是**只留最大数、其余抹零**；配票司的归一配法，是**把差距变成比例**。前者答『谁第一』，后者答『各门各占几成可能』。」

沈括问：「若差距本小，归一后北门仍占五十八票、东门五票——旅人岂不仍挤北门？」

周翁答：「会。故司里还有**温度**一说。」

---

指数台旁，另有两只**温炉**：**冷炉**与**热炉**。

同一组原始分，经冷炉时，指数台吐出的七数彼此**接近**——北门也许只得二十五票，东门十八票，西门十二票……差距被**抹平**，七门皆能分到可观之票，适合**初到百票、尚不确定哪门真宜**的日子。

经热炉时，吐出之数**极悬殊**——北门九十五票，其余五票零头，几乎等同硬配，适合**了望台极有把握、须快决**的日子。

沈括悟：冷炉使配票**均匀**，热炉使配票**尖锐**；同一原始分，换炉即换格局——**不是了望台改分，是配票司改『敢不敢把票押给第一名』**。

里正称冷炉为**探索之温**，热炉为**决断之温**。百票镇平日用温炉居中，遇节庆货潮则略加热，遇新路试开则略冷却。

---

又一日，镇外来一位**独标司**的吏员，姓韩，主张另一种配法：「七门各报原始分，我不管正负，一律**平分给每门约十四票**，再微调——公平。」

沈括请韩吏员与配票司同题竞技：根据当日子了望台评票，预测**哪一门当日实际通行量最大**——这是百票镇真正的考题，不是配票本身，而是**猜对主门**。

韩吏员平配，七门皆约十四票——猜「主门」时只能瞎指，准确率近随机。

沈括用归一配票，主门票额常达四十至七十，次门亦有十至二十——**猜主门**时，取票额最高者，准确率远高于平配。

韩吏员不服：「你北门给五十八票，东门只给五票——若实际主门是东门，你错得离谱。」

沈括摊开上月账册：归一配法**并非每票皆中**，但在千次预测中，取得票额最高之门为主门的次数，远多于平配与硬配——**错时错在『过于自信』，对时赢在『敢把多数票给高分门』**。

周翁补一句：「平配把信息**抹匀**；硬配把信息**截断**；归一配票**保留全部七门的相对关系**，又满足『票为正、和为百』——这是百票镇的规矩，也是**多门择一**时最常用的配法。」

---

沈括渐熟，里正方说破配票司在镇中的真名：

百票镇，是**一类抉择**的隐喻——不是七条物理上的门，而是**七个互斥的选项**；了望台的原始分，是**尚未归一化的得分**；指数台，是对得分做**指数变换以拉开差距**；冷炉与热炉，是**温度参数**；百票之和为一百，是**概率之和为一**。

「旅人问：今日主门是哪一扇？」沈括答：「我不必只指一扇——我给出七扇门的**成数**；成数最高者，即最可能之主门。若东门得五票、北门得五十八票，我的意思是：**北门可能性最大，但东门并非绝无可能**——除非热炉全开，或硬配派掌权。」

里正点头：「你做的，便是把七个原始分，变成七份**和为一的成数**；在多门之中**择一**时，这套配法称为——」

沈括接口：「**Softmax 配票**。」

周翁笑：「司里旧名『归一配票』；外乡人叫它 **Softmax**。名异理同。」

## 概念解释

### 这一讲真正要讲的是：Softmax 与多分类

**Softmax** 是分类任务里，把模型输出的**原始分数（logits）** 变成 **K 个类别概率** 的标准方法。概率非负，且 **K 个概率之和恒为 1**——对应故事中「百票为正、七门之和为一百」。

#### 1. 为什么需要 Softmax？

分类器最后一层常输出一个长度为 K 的向量，每个元素是对应类别的**未归一化得分** \(z_i\)（logit）：

- 可正可负，可大可小；
- 不能直接当概率（可能为负，且总和不必为 1）。

Softmax 做两件事：

1. **指数变换** \(e^{z_i}\)：保证为正，并**放大**高分与低分的差距（对应「指数台」）；
2. **归一化**：\(p_i = \dfrac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}\)，使 \(p_i \ge 0\) 且 \(\sum_i p_i = 1\)（对应「配票归一」）。

公式：

\[
\text{Softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}
\]

#### 2. 多分类（Multi-class Classification）

**多分类**：每个样本只属于 **K 个互斥类别中的一个**（百票镇「七门择一入镇」）。

- 标签常用 **One-Hot** 表示：真实类别为 1，其余为 0（「主门独得确认，其余为零」）；
- 模型输出 Softmax 概率向量 \((p_1, \ldots, p_K)\)；
- **预测类别**：取概率最大者 \(\arg\max_i p_i\)（对应「硬配派」的极端，但训练时仍用 Softmax 分布）。

与**多标签分类**不同：多标签中一个样本可同时属于多个类；多分类中**恰选一个**。

#### 3. 温度（Temperature）

带温度参数 \(T\) 的 Softmax：

\[
p_i = \frac{e^{z_i / T}}{\sum_j e^{z_j / T}}
\]

- **\(T \to 0\)**（热炉）：分布趋近 one-hot，最大类概率 → 1（接近 argmax / 硬配）；
- **\(T = 1\)**：标准 Softmax；
- **\(T \to \infty\)**（冷炉）：分布趋近均匀（接近「平配派」）。

推理、蒸馏、采样（如 LLM 的 temperature）中常用。

#### 4. 与损失函数、词嵌入的关系（逐步深入）

| 已学概念 | 与本讲关系 |
|----------|------------|
| **词嵌入** | 把词变成向量；分类头在向量上算 logits，再经 Softmax 得「下一词是词表哪一词」的概率——语言模型本质是超大**多分类**。 |
| **损失函数** | 多分类常用 **交叉熵（Cross-Entropy）**：衡量 Softmax 输出分布与 One-Hot 真标签的差距；与「配票成数 vs 实际主门」的错账同源。 |
| **神经网络** | 最后一层 `Linear → Softmax`（或 `Linear` + `CrossEntropyLoss` 内嵌 Softmax）是分类网络标配。 |
| **梯度下降** | 通过交叉熵对 logits 求导，反向更新前面各层——Softmax 与交叉熵组合求导有简洁形式。 |

#### 5. Softmax vs Argmax

| | Softmax | Argmax |
|---|---------|--------|
| 输出 | K 个概率，和为 1 | 单个类别索引 |
| 可微 | 是（训练用） | 否（不可导） |
| 信息 | 保留类间相对差距 | 只保留「谁第一」 |

训练用 Softmax + 交叉熵；部署预测时常对 Softmax 输出取 argmax。

#### 6. 数值稳定

直接算 \(e^{z_i}\) 在 \(z_i\) 很大时会溢出。实现常用**减最大值技巧**：

\[
\text{Softmax}(z_i) = \frac{e^{z_i - \max(z)}}{\sum_j e^{z_j - \max(z)}}
\]

数学等价，数值更安全——配票司里相当于「七门同减最高分再入指数台」，比例不变。

#### 7. 常见误区

1. **「Softmax 输出就是模型置信度」** — 未必；过度自信（miscalibration）很常见，需校准（calibration）。
2. **「类别不平衡时 Softmax 仍公平」** — 需配合类别权重、重采样或 Focal Loss 等。
3. **「Softmax 只用于分类」** — 注意力权重、对比学习中的相似度归一化，也常用 Softmax 思想（在某一维上归一化为概率）。

## 学习资料与了解途径

### 入门（本科基础）

1. **吴恩达《深度学习专项》课程 2 — Softmax 回归**  
   - 从二分类 logistic 推广到多分类 Softmax；One-Hot 标签与交叉熵。  
   - 搜索：`Andrew Ng Softmax Regression`

2. **3Blue1Brown《Softmax 与交叉熵》相关章节**（Neural Networks 系列）  
   - 可视化指数、归一化与梯度；适合建立直觉。

3. **PyTorch 官方教程 — 训练分类器（Training a Classifier）**  
   - 网址：https://pytorch.org/tutorials/beginner/blitz/cifar10_tutorial.html  
   - CIFAR-10 十类分类，最后一层 logits + `CrossEntropyLoss` 即 Softmax 多分类标准写法。

### 教材与课程章节

4. **《深度学习》（花书）第 6.2.2 节 — Softmax 单元**  
   - 形式化定义、与 logistic 的关系、最大似然解释。

5. **《统计学习方法》（李航）第 6 章逻辑斯谛回归**  
   - 可推广到 Softmax 回归（多类 logistic）；国内本科常用教材。

6. **CS231n（Stanford）Lecture 2 — Linear Classification**  
   - 网址：http://cs231n.stanford.edu/  
   - Softmax、SVM、交叉熵对比；作业实现 multi-class linear classifier。

### 代码实践

7. **NumPy 手写 Softmax + 交叉熵**  
   ```python
   import numpy as np
   def softmax(z):
       z = z - z.max()  # 数值稳定
       e = np.exp(z)
       return e / e.sum()
   logits = np.array([2.1, 0.3, -1.0, 4.5, 1.2, -0.5, 0.8])  # 百票镇七门
   print(softmax(logits))  # 和为 1 的概率向量
   ```

8. **scikit-learn `LogisticRegression(multi_class='multinomial')`**  
   - 内部即 Softmax 回归；在 Iris 等数据集上对比 `ovr`（一对多）与 `multinomial`（Softmax）。

9. **PyTorch `nn.CrossEntropyLoss`**  
   - 文档：https://pytorch.org/docs/stable/generated/torch.nn.CrossEntropyLoss.html  
   - 注意：输入为 **logits**（未做 Softmax），内部含 `LogSoftmax + NLLLoss`；勿对 logits 先 Softmax 再传入。

### 进阶延伸

10. **Temperature Scaling（Guo et al., 2017）**  
    - 论文：*On Calibration of Modern Neural Networks* — 理解「配票成数 ≠ 真实把握」与温度校准。

11. **语言模型中的 Softmax**  
    - 词表大小为 K 的下一词预测，是 **超大 Softmax**；采样时的 temperature、top-k、top-p 都是在 Softmax 概率上再做文章——与「冷炉 / 热炉」直接对应。

12. **Attention is All You Need**  
    - 自注意力里对 score 做 Softmax 得到权重，和为 1 — 同一数学工具，用于「在多个位置间分配注意力百票」。

### 实践建议

- 用百票镇七门原始分，手算或编程 Softmax，验证和为 1；再试 \(T=0.5, 1, 2, 10\)，观察分布变尖或变平。  
- 在 MNIST / Fashion-MNIST 上训练一层 Softmax 回归，对比 **argmax 预测** 与 **Softmax 概率** 的差异。  
- 画 **校准曲线（reliability diagram）**：模型说 70% 把握时，是否真的约 70% 正确——检验「配票成数是否诚实」。  
- 结合已学 **词嵌入**：在 small text 语料上建 `Embedding → Linear(vocab_size) → CrossEntropyLoss`，理解 NLP 分类与语言模型的统一框架。
